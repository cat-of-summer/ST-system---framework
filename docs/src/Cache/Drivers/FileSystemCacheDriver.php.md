# FileSystemCacheDriver.php
<!-- DOCGEN:END -->

`ST_system\Cache\Drivers\FileSystemCacheDriver` — драйвер кеша поверх обычной файловой
системы. Это **драйвер по умолчанию** (`drivers.default => 'filesystem'` в `CacheManager`) и
именно на него автоматически падает фолбэк, если запрошенный драйвер недоступен, поэтому его
можно считать «гарантированно рабочим» бэкендом.

## Выбор через конфиг

```php
CacheManager::make($key, [
    'driver' => 'filesystem', // либо не указывать вовсе — это драйвер по умолчанию
    'dir'    => '~/cache/my-feature/',
    'file'   => 'data',
    'ttl'    => 3600,
]);
```

- `dir` — базовая директория кеша (по умолчанию `CacheManager::config('default.dir')`).
  Разворачивается в абсолютный путь через `Main::preparePath($dir, 3)` (поддерживает `~`).
- `file` — имя блоба по умолчанию (см. мульти-файловость в `CacheDriver.php.md`).
- `ttl` — TTL по умолчанию (секунды), `-1` — бессрочно.

## Устройство хранилища

Каждый ключ кеша получает свою поддиректорию `{dir}/{hash(key)}/`. Внутри неё блоб данных
лежит как файл `{file}`, а мета — как `{file}.meta` (JSON). `isAvailable()` создаёт базовую
директорию при необходимости (`mkdir(..., 0775, true)`) и проверяет, что она существует и
доступна на запись; результат мемоизируется на процесс в статическом `self::$initialized[$dir]`,
чтобы не дёргать `is_dir`/`is_writable` на каждое обращение.

### Блокировки и атомарность

- `writeBlob()` пишет файл данных под `LOCK_EX` (`file_put_contents(..., LOCK_EX)`).
- `readBlob()` открывает тот же файл и берёт `LOCK_SH` прямо на нём (без отдельного
  sidecar-лока) — это не плодит лишние файлы на каждое чтение и корректно координируется с
  `LOCK_EX` из `writeBlob()` на одном и том же дескрипторе.
- `writeMeta()` — атомарная запись меты: эксклюзивная блокировка на `{file}.meta.lock`,
  запись во временный файл `{file}.meta.tmp.<random>` и `rename()` поверх `{file}.meta`.
  Благодаря `rename()` читатель никогда не увидит частично записанный JSON.
- `readMeta()` берёт `LOCK_SH` прямо на `.meta`-файле (без sidecar-лока) — атомарность
  записи через tmp+rename гарантирует, что читатель всегда видит либо старую, либо новую
  целую версию.

### Удаление

- `purgeStorage()` — рекурсивно удаляет всю поддиректорию текущего ключа (под защитой
  директорийного `.lock`-файла на время удаления).
- `purgeBaseStorage()` — рекурсивно очищает всю базовую директорию `dir` (все ключи всех
  ключей этого пространства).
- `purgeExpiredStorage()` / `purgeExpiredBaseStorage()` — сканируют `*.meta` файлы
  (`glob`), парсят `expires_in` и удаляют пары блоб+мета только для просроченных записей
  (`expires_in !== -1 && expires_in < time()`), не трогая ещё валидные.

`initDir()` — публичный метод, инициализирует и мемоизирует конкретную директорию; используется
также в `spawn()`-сценариях (`__rebind()` пересчитывает `dir` при смене `dir` без пересоздания
драйвера).
