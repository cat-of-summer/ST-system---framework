# FileSystemCacheDriver


## 1. Концепция

`FileSystemCacheDriver` — конкретный кэш-драйвер, хранящий данные в файловой системе. Каждый ключ кэша получает собственную директорию `<base_dir>/<md5(key)>/`, внутри которой создаются файлы блобов и `.meta`-файлы.

**Особенности:**

- Параллельный доступ защищён файловыми блокировками (`flock`). Запись блобов идёт через атомарный `rename` временного файла.
- Метаданные хранятся как JSON-файлы с расширением `.meta`. Блоб при этом необязателен: слот, у которого есть только `.meta`, — законный способ хранить маленькое значение одним файлом вместо двух (так делает `ImageMime::getImageSize()`).
- `isAvailable()` создаёт базовую директорию при первом вызове (идемпотентно, через `static::$initialized`).
- `purge(true)` удаляет директорию ключа; `purgeBase(true)` рекурсивно удаляет весь базовый каталог. `purge(false)` только чистит кэши в памяти.
- Путь поддерживает `~`-префикс (раскрывается через `Main::preparePath`).

> **Важно для долгоживущих процессов.** Результат `initDir()` запоминается в статическом `$initialized` на весь процесс. Все четыре метода `purge*Storage()` удаляют оттуда записи об удалённых директориях — иначе после `purge()` драйвер считал бы директорию существующей, и следующий `writeMeta()` упал бы с `RuntimeException` («Cannot open lock file»), потому что `.lock` некуда положить.

```php
// Прямое использование
$cache = new FileSystemCacheDriver('user:42', [
    'dir' => '~/cache/',
    'ttl' => 3600,
]);
$cache->set(['name' => 'Ivan']);
echo $cache->get()['name']; // Ivan
```

Чаще всего используется через `Manager`:

```php
Manager::setConfig(['drivers' => ['filesystem' => ['dir' => '/var/cache']]]);
$data = Manager::get('user:42'); // использует FileSystemCacheDriver по умолчанию
```

## 2. Конфигурация

| Ключ | По умолчанию | Описание |
|---|---|---|
| `dir` | `'~/cache/'` | Базовая директория хранилища |
| `file` | `'data'` | Имя файла блоба по умолчанию |
| `ttl` | `3600` | TTL в секундах (0 = из конфига, -1 = вечно) |

## 3. Публичные методы

### `isAvailable(): bool`

Проверяет, доступна ли и записываема ли базовая директория. При первом вызове создаёт её рекурсивно (`mkdir(..., true)`).

```php
if (!$cache->isAvailable()) {
    // директорию создать не удалось
}
```

---

### `initDir(string $dir = ''): bool`

Создаёт директорию (рекурсивно) и проверяет, что она существует и доступна на запись. Результат мемоизируется в `static::$initialized` по пути. Записи оттуда удаляются при `purge*()`.

---

## 4. Раскладка на диске

```
<base_dir>/
  <md5(key)>/
    data              ← блоб (имя из 'file')
    data.meta         ← метаданные блоба, JSON
    photo.imagesize.meta   ← слот только с метаданными, без блоба
```

`purgeExpired()` / `purgeExpiredBase()` обходят `*.meta`, читают `expires_in` и удаляют пару «блоб + мета», если срок истёк. **Мета без `expires_in` считается истёкшей** (`?? 0`), поэтому долгоживущие записи нужно писать с `ttl = -1` — тогда `expires_in === -1` и сборщик их пропускает.

---

Остальные публичные методы наследуются от `CacheDriver`: `set`, `get`, `setMeta`, `getMeta`, `exists`, `isExpired`, `isValid`, `purge`, `purgeBase`, `purgeExpired`, `purgeExpiredBase`, `spawn`.
