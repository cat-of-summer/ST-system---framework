# MemcacheExtAdapter.php
<!-- DOCGEN:END -->

`class MemcacheExtAdapter implements MemcachedAdapterInterface` (`ST_system\Cache\Drivers\Memcached`) — адаптер для устаревшего PECL-расширения **`memcache`** (класс `\Memcache`, без "d" на конце — не путать с `\Memcached`). Оставлен для окружений, где по каким-то причинам доступно только старое расширение. `isAvailable()` проверяет `class_exists(\Memcache::class)`.

## Подключение

```php
[
    'servers' => [
        ['host' => '10.0.0.1', 'port' => 11211],
        // либо индексированной формой: ['10.0.0.2', 11211]
    ],
    // либо один сервер вместо 'servers':
    'host' => '127.0.0.1',
    'port' => 11211,
]
```

Каждый сервер из `servers` (или единственный `host`/`port`) добавляется через `addServer($host, $port)`. Если серверов нет ни в `servers`, ни в `host` — `\RuntimeException`. В отличие от `MemcachedExtAdapter`, здесь **нет** `persistent_id` и **нет** SASL-аутентификации — легаси-расширение `memcache` этого не поддерживает.

## Особенности реализации

- `set()` вызывает `\Memcache::set($key, $value, 0, $expiry)` — третий аргумент (`0`) отключает флаг сжатия `MEMCACHE_COMPRESSED`, значение всегда пишется без сжатия.
- `touch()` **эмулируется**: в самом расширении `memcache` нет нативного метода `touch()` (в отличие от `memcached`/Redis), поэтому адаптер читает текущее значение и, если оно есть, перезаписывает его тем же значением с новым `expiry` — это два round-trip'а вместо одного и не атомарно (между чтением и записью значение может смениться).
- `exists()` реализован через `get() !== false` — как и `touch()`, это накладывает то же ограничение: невозможно отличить реально хранимое значение `false` от отсутствия ключа.
- `delete()` перебирает переданные ключи по одному (`\Memcache` не имеет пакетного удаления, в отличие от `deleteMulti()` у `\Memcached`).
