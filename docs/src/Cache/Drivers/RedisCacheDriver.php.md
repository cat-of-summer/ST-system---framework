# RedisCacheDriver.php
<!-- DOCGEN:END -->

`ST_system\Cache\Drivers\RedisCacheDriver` — драйвер кеша поверх Redis. Сам драйвер не завязан
на конкретную клиентскую библиотеку: он делегирует операции объекту, реализующему
`ST_system\Cache\Drivers\Redis\RedisAdapterInterface`, и автоматически выбирает первый доступный
адаптер из `PhpRedisAdapter` (расширение `redis`) и `PredisAdapter` (библиотека Predis) — они
лежат в поддиректории `Redis/` и документируются отдельно.

## Выбор через конфиг

```php
CacheManager::make($key, [
    'driver' => 'redis',
    'host'   => '127.0.0.1',
    'port'   => 6379,
    'auth'   => null,
    'db'     => 0,
    'prefix' => 'cache:',
    'ttl'    => 3600,
]);

// либо готовое соединение (адаптер) напрямую:
CacheManager::make($key, ['driver' => 'redis', 'connection' => $adapter]);
```

## Пул соединений и disconnect()

Соединения кешируются в статическом пуле процесса по хэшу параметров подключения
(`host`/`port`/`auth`/`db`). `RedisCacheDriver::disconnect()` роняет все пуловые соединения
процесса (кроме внедрённых извне через `connection`) — **обязателен к вызову в дочернем
процессе сразу после `pcntl_fork()`**: потомок наследует сокет родителя, и если оба начнут
писать в один дескриптор, их байтовые потоки перемешаются, что выглядит как случайные,
невоспроизводимые ошибки Redis. Реальное закрытие сокетов происходит в деструкторах адаптеров.

## Хранение данных

В отличие от Memcached, Redis умеет и структуры данных, и перечисление ключей, поэтому:

- Каждый ключ кеша — это Redis HASH с именем `bucket = prefix . hash(key)`; блоб и мета
  хранятся как поля этого хеша (`file` и `file.meta`) через `hSet`/`hGet`/`hExists`.
- `purgeStorage()` удаляет весь хеш целиком (`del($bucket)`).
- `purgeBaseStorage()` и `purgeExpiredBaseStorage()` используют курсорное `SCAN`
  (`scan($cursor, prefix.'*', 500)`), чтобы найти все бакеты этого драйвера и удалить либо
  все, либо только с просроченным `expires_in` в мете — то есть, в отличие от Memcached,
  здесь доступна полноценная точечная очистка по префиксу без сброса всего сервера.
