# Main.php
<!-- DOCGEN:END -->

## Что это

`ST_system\Main` — финальный класс-контейнер статических util-хелперов общего назначения, без состояния экземпляра (у него нет и не предполагается создание объектов). Это «стандартная библиотека» фреймворка: строковые преобразования регистра (`studlyCase`/`camelCase`/`snakeCase`/`kebabCase`), работа с путями (`preparePath`), dot-notation доступ к вложенным массивам (`dotGet`/`dotSet`/`dotFlatten`), детерминированное хеширование произвольных значений (`hash`), генерация UUID, глубокое слияние массивов/конфигов (`merge`), форматирование размеров в байтах (`formatBytes`) и т.д. Практически каждый класс фреймворка (`View`, `Assets`, `Access`, `Debug`, `Storage\File`, `Cache\*`, `Schemas\*`, `API\*` и другие) использует `Main` как общий низкоуровневый инструментарий — в частности, для построения ключей кеша, путей к директориям кеша и нормализации конфигов.

## Публичные методы

### `Main::timestamp(string $format = ''): string`

Без аргумента возвращает текущий момент времени в виде строки с высокой точностью — секунды с дробной частью, полученные из `hrtime(true) / 1e9` (если `hrtime()` доступна — монотонные, не зависящие от перевода системных часов), либо из `microtime(true)`. Используется как «тик» таймера: два таких значения можно вычесть друг из друга (PHP автоматически приведёт строки к числам), получив длительность в секундах.

Если передан `$format` (непустая строка) — метод игнорирует hi-res часть и возвращает **обычную** дату/время текущего момента (`time()`), отформатированную через `DateTime::format($format)` (стандартные символы формата PHP: `Y-m-d H:i:s` и т.п.).

```php
use ST_system\Main;

// Замер длительности операции
$start = Main::timestamp();
// ... код ...
$elapsed = (float)Main::timestamp() - (float)$start; // секунды

// Форматированная дата
echo Main::timestamp('Y-m-d H:i:s'); // например: 2026-07-14 12:30:00
```

### `Main::pluralForm($n, $forms): string`

Выбирает нужную форму множественного числа для русского языка. `$n` приводится к `int`, `$forms` — массив из **трёх** строк: `[форма_для_1, форма_для_2_4, форма_для_5_и_дальше]` (стандартные русские правила: 1 — «файл», 2 — «файла», 5 — «файлов», с учётом исключений 11–14).

```php
echo Main::pluralForm(1, ['файл', 'файла', 'файлов']);  // "файл"
echo Main::pluralForm(3, ['файл', 'файла', 'файлов']);  // "файла"
echo Main::pluralForm(11, ['файл', 'файла', 'файлов']); // "файлов"
echo Main::pluralForm(21, ['файл', 'файла', 'файлов']); // "файл"
```

### `Main::basename(string $class): string`

Возвращает «короткое имя» — последний сегмент после `\` (или `/`, который предварительно заменяется на `\`) в переданной строке. Чаще всего используется для получения короткого имени класса из FQCN (`static::class`), но так же работает и для обычных путей с `/`.

```php
Main::basename('ST_system\HTTP\Route'); // "Route"
Main::basename('a/b/c');                // "c"
```

---

### `Main::glue(array $parts, string $separator): string`

Обрезает `$separator` с обоих концов у каждого элемента `$parts`, отбрасывает получившиеся пустые элементы и склеивает оставшиеся через `$separator`. Удобно для безопасной сборки путей/составных ключей из частей, у которых неизвестно, есть ли на концах разделитель.

```php
Main::glue(['/cache/', '/views/', 'Home'], '/'); // "cache/views/Home"
Main::glue(['', 'a', null ?? '', 'b'], '/');     // "a/b"
```

Типичное применение в фреймворке — построение пути к директории кеша конкретного класса:

```php
'dir' => Main::glue([CacheManager::config('default.dir'), Main::basename(static::class)], '/'),
```

### `Main::studlyCase(string $name): string`

Преобразует строку в `StudlyCase` (он же `PascalCase`): разбивает исходную строку на «слова» по пробелам/дефисам/подчёркиваниям и по границам camelCase (нижний-верхний регистр, а также акронимы вроде `HTTPServer` → `HTTP`+`Server`), затем каждое слово переводится в нижний регистр и получает заглавную первую букву. Результат кешируется по входной строке (статический массив на процесс).

```php
Main::studlyCase('user_profile-name'); // "UserProfileName"
Main::studlyCase('httpServer');        // "HttpServer"
```

---

### `Main::camelCase(string $name): string`

То же самое, что `studlyCase()`, но с маленькой первой буквой (`lcfirst(studlyCase($name))`). Используется, например, в `Traits\HasAttributes` для построения имён методов-аксессоров (`get{CamelCase}Attribute`).

```php
Main::camelCase('user_profile_name'); // "userProfileName"
```

---

### `Main::snakeCase(string $name): string`

Преобразует строку в `snake_case` (слова, определяемые тем же алгоритмом разбиения, что и в `studlyCase()`, приводятся к нижнему регистру и склеиваются через `_`). Кешируется по входной строке.

```php
Main::snakeCase('UserProfileName'); // "user_profile_name"
```

---

### `Main::kebabCase(string $name): string`

Аналогично `snakeCase()`, но со склейкой через `-`.

```php
Main::kebabCase('UserProfileName'); // "user-profile-name"
```

### `Main::merge(...$arrays)`

Глубокое (рекурсивное) слияние произвольного числа массивов слева направо: значения из более поздних массивов побеждают, за исключением случая, когда значение в обоих массивах — массив (тогда они мержатся рекурсивно) или когда значение в обоих массивах — `callable`/`Closure` (тогда результатом становится **новый** `Closure`, который при вызове с теми же аргументами вызовет оба исходных callable и глубоко смержит их результаты — то есть слияние «ленивых» значений откладывается до фактического вызова).

```php
$base = ['cache' => ['driver' => 'filesystem', 'ttl' => 60]];
$override = ['cache' => ['driver' => 'redis']];

Main::merge($base, $override);
// ['cache' => ['driver' => 'redis', 'ttl' => 60]]

$a = fn() => ['x' => 1];
$b = fn() => ['x' => 2, 'y' => 3];
$merged = Main::merge(['fn' => $a], ['fn' => $b]);
$merged['fn'](); // ['x' => 2, 'y' => 3] — вычисляется только при вызове
```

Используется, в частности, во `View` для слияния слоёв конфигурации (`Main::merge(static::config(), $sourceConfig, $this->parentBundle, $this->override)`).

### `Main::hash($value): string`

Возвращает детерминированный MD5-хеш содержимого произвольного значения PHP: строки, числа (int/float с фиксированной точностью `%.14G`), bool, `null`, массивы (в том числе вложенные), `Closure` (по имени файла, диапазону строк и захваченным static-переменным) и объекты (по классу и приведению к массиву/`JsonSerializable::jsonSerialize()`, с обнаружением циклических ссылок через `spl_object_hash`).

Ключевая особенность — **независимость от порядка**: элементы списков (`array`, где ключи `0..n-1` по порядку) перед хешированием сортируются как строки, а ключи ассоциативных массивов сортируются по имени/типу — то есть одинаковые по содержанию, но по-разному упорядоченные массивы дают одинаковый хеш. Для неподдерживаемых типов (например, `resource`) бросает `\RuntimeException`.

```php
Main::hash(['b' => 2, 'a' => 1]) === Main::hash(['a' => 1, 'b' => 2]); // true
Main::hash([1, 2, 3]) === Main::hash([3, 2, 1]);                       // true (список, порядок игнорируется)

// Типичное применение — построение ключа кеша
$key = Main::hash([$file->getPathname(), $attrs]);
```

### `Main::uuid(int $version = 7): string`

Генерирует UUID. По умолчанию (`$version = 7` либо любое значение, кроме `4`) — **UUIDv7**: время-упорядоченный идентификатор на основе 48-битной метки времени в миллисекундах плюс случайные биты, с корректно выставленными битами версии/варианта. При `$version = 4` генерирует классический случайный **UUIDv4**.

```php
Main::uuid();     // например: "0189f3a2-7b1c-7e4d-8a2f-1234567890ab" (UUIDv7)
Main::uuid(4);     // случайный UUIDv4
```

Применяется, например, во `View` как соль кеша (`'salt' => Main::uuid()`).

### `Main::dotGet(array $data, string $path, $default = null)`

Читает вложенное значение массива `$data` по dot-пути `$path` (например, `'cache.driver'`). Каждый сегмент пути ищется как строковый ключ, а если такого нет — как целочисленный (для числовых сегментов вроде `'0'`, `'1'`, что позволяет адресоваться и в списки). Если по пути встретился не-массив или сегмент не найден — возвращается `$default`.

```php
$data = ['cache' => ['driver' => 'redis', 'hosts' => ['redis1', 'redis2']]];

Main::dotGet($data, 'cache.driver');      // "redis"
Main::dotGet($data, 'cache.hosts.1');     // "redis2"
Main::dotGet($data, 'cache.missing', 'x'); // "x"
```

---

### `Main::dotSet(array &$data, string $path, $value): void`

Записывает значение `$value` во вложенный массив `$data` по dot-пути `$path`, по ходу создавая недостающие промежуточные массивы (и перезаписывая ими любые не-массивы, встретившиеся на пути).

```php
$data = [];
Main::dotSet($data, 'cache.driver', 'redis');
// $data === ['cache' => ['driver' => 'redis']]
```

---

### `Main::dotFlatten(array $data, string $prefix = ''): array`

«Расплющивает» вложенный ассоциативный массив в плоский массив вида `['a.b.c' => value]`. Важный нюанс: **непустые списковые** массивы (последовательные числовые ключи `0..n-1`, см. `arrayIsList()`) не разворачиваются рекурсивно, а остаются атомарным значением-листом целиком — это сделано намеренно, чтобы при последующем `dotSet()`/слиянии список-override заменял старый список целиком, а не мержился поэлементно.

```php
Main::dotFlatten([
    'cache' => ['driver' => 'redis', 'hosts' => ['h1', 'h2']],
    'debug' => true,
]);
// [
//   'cache.driver' => 'redis',
//   'cache.hosts'  => ['h1', 'h2'], // список — не разворачивается
//   'debug'        => true,
// ]
```

---

### `Main::arrayIsList(array $arr): bool`

Проверяет, является ли массив «списком» — то есть его ключи строго последовательны и начинаются с `0` (`0, 1, 2, ...`). Аналог `array_is_list()` из PHP 8.1+, реализованный вручную для совместимости.

```php
Main::arrayIsList([1, 2, 3]);        // true
Main::arrayIsList(['a' => 1]);       // false
Main::arrayIsList([1 => 'a', 0 => 'b']); // false (порядок ключей нарушен)
```

### `Main::formatBytes($bytes, string $format = '', int $precision = 2)`

Форматирует количество байт `$bytes` в человекочитаемый вид. Использует константу `Main::BYTE_UNITS` — таблицу множителей от `pb` (петабайт, `2^50`) до `b` (байт, `1`).

- **Без `$format`** (по умолчанию): подбирает наибольшую единицу, в которую `$bytes` помещается (перебор от `pb` вниз до `b`), и возвращает строку вида `"<округлено до $precision знаков> <ЕДИНИЦА>"`.

```php
Main::formatBytes(1572864);       // "1.5 MB"
Main::formatBytes(500);           // "500 B"
Main::formatBytes(1572864, '', 0); // "2 MB"
```

- **С `$format`**: мини-шаблон, где распознаются токены единиц (`b`, `kb`, `mb`, `gb`, `tb`, `pb`, а также трёхбуквенные варианты вроде `kib`/`mib`, приводимые к тем же единицам), не являющиеся частью токена символы — литеральный текст (`\` экранирует следующий символ, чтобы вставить его буквально, даже если это похоже на токен единицы).
  - Если в `$format` **ровно один** токен единицы и весь формат состоит только из него (без окружающего литерального текста) — возвращается **число** (не строка): `int`, если единица — байты, иначе `float` (результат деления).
  - Если единиц несколько (или есть литеральный текст вокруг токена) — метод работает как «разрядный» форматтер длительности: последовательно нацело делит остаток байт на каждую единицу по убыванию и склеивает части с литеральным текстом.

```php
Main::formatBytes(1572864, 'mb');        // 1.5 (float)
Main::formatBytes(1572864, 'kb');        // 1536 (float, т.к. kb !== 'b')
Main::formatBytes(500, 'b');             // 500 (int)

Main::formatBytes(1610612736, 'gb"GB "mb"MB"'); // "1GB 512MB" (пример разрядного форматирования)
```

### `Main::preparePath(string $path, $base = 0, bool $strict = false): string`

Нормализует путь: разрешает относительные пути относительно базовой директории, схлопывает сегменты `.`/`..` и возвращает абсолютный путь в unix-стиле (с одним ведущим `/`, без завершающего `/`).

Логика определения базовой точки в зависимости от вида `$path`:
- `$path` начинается с `~` — база берётся из `Config::env('DOCUMENT_ROOT')`, а если она пуста — из `Config::env('COMPOSER_ROOT')`; символы `/` и `~` обрезаются с обоих концов остатка пути.
- `$path` не начинается с `/` (обычный относительный путь) — база определяется параметром `$base`:
  - если `$base` — `int` (по умолчанию `0`), берётся файл из `debug_backtrace()` на глубине `$base` и используется директория этого файла (`dirname(...)`) — то есть **директория вызывающего кода**. Значение `0` означает «файл, который непосредственно вызвал `preparePath()`»; большие значения позволяют подняться выше по стеку вызовов (например, `1` — вызывающий вызывающего).
  - если `$base` — строка, она используется как базовая директория напрямую.
- `$path` начинается с `/` — уже абсолютный путь, используется как есть.

После определения полного пути он построчно (по `/`) схлопывается: пустые сегменты и `.` пропускаются; `..` либо (по умолчанию) выталкивает последний сегмент из стека, либо (если `$strict = true`) бросает `\InvalidArgumentException`, если встречается недопустимый выход за пределы (защита от path traversal в «строгом» режиме).

```php
use ST_system\Main;

// Относительно директории текущего файла
$path = Main::preparePath('templates/home.php');

// Относительно явной директории
$path = Main::preparePath('../cache', __DIR__);

// Относительно корня проекта/DOCUMENT_ROOT
$path = Main::preparePath('~/storage/logs');

// Строгий режим — запрещает '..' в пользовательском вводе
$path = Main::preparePath($userSuppliedPath, __DIR__, true); // бросит исключение при '../../etc/passwd'
```

### `Main::BYTE_UNITS`

```php
public const BYTE_UNITS = [
    'pb' => 1125899906842624,
    'tb' => 1099511627776,
    'gb' => 1073741824,
    'mb' => 1048576,
    'kb' => 1024,
    'b'  => 1,
];
```

Таблица множителей байт для `formatBytes()` (степени 1024). Публичная — доступна для использования извне.
