# Lang.php
<!-- DOCGEN:END -->

`final class Lang` — локализация, гибрид подходов Bitrix и Laravel: фразы хранятся не в одном большом файле на локаль, а рассыпаны по дереву каталогов источника (файлы `<локаль>.php`, возвращающие массив, и/или `<локаль>.json`), а ключ обращения "переливается" между путём файловой системы и путём внутри PHP/JSON-массива в любой точке разбиения. Поверх поиска по дереву есть компилируемый кеш (одна плоская карта на пару источник+локаль) с mtime-инвалидацией без пересканирования дерева, runtime-оверрайды в памяти процесса и Laravel-стиль плейсхолдеров/склонений (`choice()`).

Зависимости: `Traits\HasConfig` (конфиг класса), `Cache\CacheManager` (хранение скомпилированной карты), `Main` (dot-notation `dotGet`/`dotSet`/`dotFlatten`, детерминированный хеш `hash()`, `preparePath()`, `glue()`, `basename()`). Используется [View.php](View.php.md) не напрямую, а симметрично: `getUsedFiles()` — тот же паттерн хука, что и в `Assets` — существует, чтобы вызывающий код (например, кеш `View`) мог включить задействованные языковые файлы в свои собственные зависимости инвалидации.

## Конфигурация по умолчанию

```php
'source'     => ['~' => '~'],
'locale'     => 'ru',
'fallback'   => 'en',
'extensions' => ['php', 'json'],
'dir_name'   => '',
'cache' => [
    'use'     => true,
    'driver'  => 'filesystem',
    'dir'     => …,
    'check'   => 'mtime',
    'ttl'     => 60,
    'exclude' => [],
],
'plural' => [],
```

`source` — карта именованных источников фраз (см. ниже). `locale`/`fallback` могут быть строкой, либо `callable`/`\Closure` — тогда значение резолвится вызовом при каждом обращении (`resolveLocaleValue()`), что удобно для локали, зависящей от текущего запроса. `extensions` — расширения файлов фраз в порядке приоритета (при конфликте в одном каталоге побеждает то, что стоит раньше в списке). `dir_name` — имя подкаталога, где лежат языковые файлы (пусто — файлы лежат прямо в дереве источника вперемешку с остальным содержимым, например компонентами). `cache.check` — режим проверки актуальности компилированной карты (`'mtime'` / `'ttl'` / `'never'`, см. `compiledMap()` ниже). `cache.exclude` — пути/имена источников/regex, которые кеш обходит стороной и всегда читает "живьём". `plural` — карта `локаль => callable(int $n): int` для кастомных правил склонения, переопределяющих встроенные.

Конфиг класса иммутабелен в том же смысле, что и у других классов на `HasConfig`: смена языка **во время работы** должна идти через `setLocale()`, а не через повторный `setConfig(['locale' => ...])` — второе меняет глобальный дефолт, а не runtime-выбор конкретного запроса.

## Источники (`source`)

Как и в `View`, `config('source')` описывает набор именованных источников: `['~' => '~', 'admin' => '~/admin/lang']`. Обращение к неглавному источнику — либо явным префиксом ключа через `:` (`Lang::get('admin:menu.title')`), либо синтаксическим сахаром через `__callStatic` (`Lang::admin('menu.title')` — идентично `Lang::get('admin:menu.title')`, все переданные аргументы после ключа перетекают в `get()` как есть). Источник `'~'` (корень) добавляется автоматически, если явно не переопределён в `source`, — поэтому `Lang::get('some.key')` без префикса работает всегда, даже если `source` переопределён и не содержит `'~'`.

Каждый источник в `spec`-форме (`['source' => путь, ...]`) может переопределить `extensions`/`dir_name` персонально для себя (`sources()['name']['ext']`/`['dir_name']`); если источник задан просто строкой-путём — используются глобальные `extensions`/`dir_name`. Источник с пустым путём (`''`) молча пропускается (не регистрируется) — способ отключить источник, не удаляя ключ из конфига.

### Зарезервированные имена

Константа `RESERVED` — `get, set, has, all, choice, config, setConfig, applyConfig, setLocale, getLocale, setFallback, getFallback, purge, getUsedFiles, sources`. Источник с именем из этого списка конфликтует со служебными методами класса и бросает `LogicException` уже при построении карты источников (`sources()`), не дожидаясь первого обращения к ключу.

### `__callStatic(string $name, array $args)`

Любое имя статического метода, не входящее в `RESERVED` и присутствующее среди зарегистрированных источников, трактуется как имя источника: `Lang::admin('menu.title', $params)` превращается в `Lang::get('admin:'.'menu.title', $params)`. Первый переданный аргумент обязателен и приводится к строке как ключ; остальные аргументы прокидываются в `get()` как `$params, $default, $locale`. Неизвестное имя (не источник и не зарезервированное) — `BadMethodCallException`.

## Разрешение ключа: перелив между ФС и массивом

Это центральная идея класса. `Lang::get('catalog.list.product.name')` находит значение во **всех** этих равнозначных раскладках:

- `/catalog/list/product/ru.php => ['name' => ...]` (самое глубокое разбиение — отдельный каталог на каждый сегмент пути);
- `/catalog/list/ru.php => ['product' => ['name' => ...]]` (частичное разбиение — остаток пути живёт внутри массива файла);
- корневой `/ru.json` с полным вложенным путём `['catalog' => ['list' => ['product' => ['name' => ...]]]]` (разбиения нет вовсе).

При конфликте между кандидатами побеждает **более глубокий (специфичный) файл**; в пределах одного каталога — расширение с более высоким приоритетом (порядок в `extensions`).

`find(string $key, string $locale)` — точка входа резолвинга одного ключа:

1. Сперва проверяется runtime-оверрайд (`self::$runtime[$locale][$key]`, точное совпадение полного ключа, включая префикс источника, если он был) — высший приоритет, минует всё остальное.
2. Ключ разбивается `splitKey()` на имя источника (часть до `:`, по умолчанию `'~'`) и путь (часть после); если путь пуст — `ABSENT`.
3. Источник должен быть зарегистрирован (`sources()`), иначе `RuntimeException`.
4. Если кеш глобально выключен (`cache.use = false`) или конкретный источник исключён (`cache.exclude`) — используется холодный путь `resolveCold()` без обращения к компилированной карте.
5. Иначе, если в `cache.exclude` вообще что-то настроено, сначала пробуется холодный путь **только** по excluded-кандидатам (`resolveCold(..., onlyExcluded: true)`) — если найдено, это значение побеждает кешированное для этого конкретного ключа (динамический/исключённый файл всегда читается свежим, даже когда остальной источник закеширован).
6. Иначе — `mapLookup()` по компилированной карте (`compiledMap()`).

### `resolveCold()` — обход дерева без кеша

Строит цепочку кандидатных каталогов от корня источника вглубь по сегментам пути (`dirs[i] = dirs[i-1].'/'.segments[i-1]`), затем перебирает точки разбиения **от самой глубокой к корню** (`$max` — либо `n-1` при `onlyExcluded`, либо ограничен первым реально существующим каталогом при обычном холодном резолве, чтобы не гонять `is_file()` по заведомо несуществующим путям). Для каждой точки разбиения — перебор расширений в порядке приоритета, файл `{dir}{dir_name-суффикс}/{locale}.{ext}`; при `onlyExcluded` кандидат пропускается, если ни каталог, ни файл не попадают под `cache.exclude`. Первое найденное значение (`Main::dotGet` по оставшимся сегментам пути внутри содержимого файла) возвращается, а сам файл помечается использованным (`self::$usedFiles`).

### `mapLookup(array $map, string $path)` — поиск по плоской компилированной карте

Три сценария, в порядке проверки:

1. **Точный ключ** — `$map[$path]` существует напрямую.
2. **Лист-массив по префиксу** — если точного ключа нет, путь укорачивается посегментно (`errors.0` → пробуем `errors`); как только находится существующий ключ-префикс, а его значение — массив (списки не дот-флаттенятся при сборке карты, см. `Main::dotFlatten`/`arrayIsList`), в него подставляется `Main::dotGet` по оставшемуся хвосту пути (`errors.0` → `$map['errors'][0]`).
3. **Поддерево** — если ни то, ни другое не сработало, из карты собирается вложенный массив по всем ключам с префиксом `path.` (эмулирует то же поведение, что дал бы холодный запрос каталога целиком, например `catalog.list.product` без указания конкретного поля).

## Кеш: компилированная карта

`compiledMap(string $sourceName, string $locale): array` — одна плоская карта (`dot-key => значение`) на пару (источник, локаль), кешируемая через `CacheManager`. Ключ кеш-записи включает не только `[класс, источник, локаль]`, но и **саму конфигурацию источника и excludes** (`self::sources()[$sourceName]`, `self::$excludes`) — значит, изменение путей/расширений источника или правил `cache.exclude` создаёт новую кеш-запись целиком, а не полагается на stamp-инвалидацию для отслеживания структурных изменений конфигурации.

Свежесть существующей записи определяется `cache.check`:

- **`'never'`** — если мета (`deps`/`stamp`) вообще есть, запись считается вечно свежей без единого `stat()`.
- **`'ttl'`** — свежа, если с последней проверки (`meta['checked']`) прошло меньше `cache.ttl` секунд; по истечении TTL стамп всё же пересчитывается и сравнивается, и при совпадении таймер `checked` обновляется заново (throttling частоты дорогих `stat()`-проверок, а не отказ от них).
- **`'mtime'`** (по умолчанию) — стамп пересчитывается и сравнивается на каждом обращении (максимальная точность ценой I/O на каждый промах memo).

При промахе — `buildMap()` полностью обходит дерево источника заново, кеш перезаписывается вместе с метой (`stamp`, `deps`, `files`, `checked`); при попадании — карта читается из кеша, а файлы из `meta['files']` всё равно помечаются использованными (`getUsedFiles()` должен отражать реально задействованные файлы независимо от того, был ли это hit кеша или полная пересборка). Внутрипроцессный memo (`self::$maps`) дополнительно исключает повторное обращение к `CacheManager` за тот же (источник, локаль) в пределах одного запроса.

### `buildMap()` — полный обход источника

BFS по дереву каталогов источника. Если `dir_name` пуст, языковые файлы могут находиться в любом каталоге дерева наравне с прочим содержимым (`isLang` истинен с самого начала обхода); если `dir_name` задан (например `'lang'`), обход помечает каталог как "языковой" только при входе в подкаталог с этим именем, и **не спускается глубже** внутрь него — файлы фраз должны лежать непосредственно в `dir_name`, без собственной вложенности.

Найденные кандидаты сортируются перед наложением на карту: по возрастанию глубины, а при равной глубине — по убыванию приоритета расширения. Поскольку каждый следующий элемент **перезаписывает** совпадающие ключи предыдущего, такой порядок гарантирует «глубокий файл побеждает мелкий, а внутри одного каталога — файл с более приоритетным расширением побеждает».

В зависимости (`deps`) собираются **и файлы, и все посещённые каталоги** — новый или удалённый файл/подкаталог меняет mtime родителя, поэтому `depStamp()` ловит структурные изменения дерева без повторного сканирования, сравнивая только mtime уже известного списка путей.

### `depStamp(array $paths): string`

`clearstatcache()`, затем карта `путь => filemtime() или 0, если файла больше нет`, хешируется через `Main::hash()`. Это единственный сигнал инвалидации: `touch` без изменения содержимого тоже инвалидирует кеш, а изменение содержимого с сохранением mtime (редкий случай ручных манипуляций с ФС) — не инвалидирует. Тот же принцип, что и в `View::depStamp()`.

## `cache.exclude` — какие пути кеш не трогает

`buildExcludes()` разбирает `cache.exclude` на три категории по форме записи:

- строка без `/`, `~`, `\` — точное **имя источника** (`'names'`) — весь этот источник помечается `excluded` в `sources()` и всегда резолвится холодным путём, без компилированной карты;
- строка, содержащая `/`, `~` или `\` — **путь-префикс** (`'prefixes'`, приводится к абсолютному через `Main::preparePath()`) — любой файл/каталог под этим путём читается вживую даже внутри некеша-исключённого источника;
- строка вида `'regex:#...#'` — **регулярное выражение** (`'regexes'`), проверяется через `preg_match` по абсолютному пути; некорректный паттерн бросает `InvalidArgumentException` уже на этапе разбора конфига.

Каталоги самого кеша (`cache.dir` этого класса и глобальный `CacheManager::config('default.dir')`) всегда принудительно добавляются в `prefixes` — иначе запись кеш-файла меняла бы mtime каталога, входящего в его же `deps`, и карта инвалидировала бы саму себя на следующем же обращении.

## Runtime-оверрайды (`set()`)

`Lang::set($key, $value = null, ?string $locale = null)` — фразы, заданные в рантайме, живут только в памяти текущего процесса (`self::$runtime`) и имеют наивысший приоритет разрешения (проверяются в `find()` раньше файловой системы и кеша). `$key` может быть массивом — тогда он дот-флаттенится (`Main::dotFlatten`) и каждый получившийся плоский ключ пишется отдельно; либо строкой — тогда это единственная пара ключ/значение. Ключи хранятся ровно в том виде, в котором переданы, включая префикс источника через `:`, если он был указан явно.

## Публичный API чтения

- **`get(string $key, array $params = [], $default = null, ?string $locale = null)`** — резолвит `$key` в текущей (или явно указанной) локали; при `ABSENT` пробует fallback-локаль (только если она задана и отличается от текущей); при повторном `ABSENT` — `$default ?? $key` (Laravel-стиль: непереведённая строка возвращает сам ключ, а не `null`, что удобно как явный визуальный признак отсутствующего перевода прямо в верстке). Если результат — строка и переданы `$params`, выполняется `strtr()`-подстановка плейсхолдеров.
- **`choice(string $key, $number, array $params = [], $default = null, ?string $locale = null): string`** — форма множественного числа. Важный нюанс: индекс формы выбирается по правилам **той локали, где реально нашлось значение** (`$used`), а не запрошенной — то есть если ключ разрешился через fallback, используются правила плюрализации fallback-локали, а не исходной. Значение-массив трактуется как готовый список форм (`array_values`); значение-строка разбивается по `|` с обрезкой пробелов (Laravel-совместимый формат `"one|few|many"`). `params` дополнительно получает `count => $number` перед подстановкой плейсхолдеров.
- **`has(string $key, ?string $locale = null): bool`** — проверка наличия без выброса — резолвит в текущей локали, затем (если применимо) в fallback.
- **`all(string $prefix = '', ?string $locale = null): array`** — собирает вложенный массив фраз послойно: `fallback ⊂ locale ⊂ runtime` (более поздний слой перезаписывает совпадающие ключи предыдущего). Excluded-пути в выборку **не входят** — они принципиально разрешаются только поштучно через `get()`/`find()`, а не массовым обходом. Runtime-ключи при пустом `$prefix` фильтруются от ключей с явным префиксом источника (`strpos($k, ':')`) — иначе `all()` без аргументов случайно захватывал бы фразы, предназначенные для других источников.

## `purge()` и `getUsedFiles()`

`purge(): void` — деплой-хук: очищает физическую кеш-запись компилированных карт (`CacheManager::make('')->purgeBase()` с конфигом `cache.driver`/`cache.dir`) и внутрипроцессные memo (`self::$maps`, `self::$fileData`). Не трогает `self::$sources`/`self::$excludes` — это память конфигурации, а не кеша фраз.

`getUsedFiles(): array` — плоский список путей всех языковых файлов, реально поучаствовавших в разрешении хоть одного ключа за время жизни процесса (независимо от того, шли они через холодный путь, полную пересборку карты или cache-hit). Существует как хук для внешнего кода — например, чтобы кеш `View`, отрендеривший шаблон с вызовами `Lang::get()`, мог добавить эти файлы в собственные зависимости инвалидации и корректно протухать при правке переводов.

## Плюрализация и плейсхолдеры

`pluralIndex(int $n, string $locale): int` — сначала проверяет `config('plural')[$locale]` (пользовательский `callable`, результат приводится к неотрицательному int); иначе встроенные правила: 3-форменная славянская схема для `ru`/`uk`/`be` (`1, 21, 31... → форма 0`; `2-4, 22-24... → форма 1`; остальное → форма 2`), для всех прочих локалей — 2-форменная схема как в английском (`1 → форма 0`, иначе `форма 1`). `$n` берётся по модулю (`abs`) — отрицательные числа склоняются по тому же правилу, что и положительные.

`placeholders(array $params): array` строит карту для `strtr()` в трёх регистровых вариантах на каждый параметр — `:name` (как есть), `:Name` (`mb`-safe первая заглавная), `:NAME` (`mb_strtoupper`) — так авторам файлов фраз не нужно заботиться о регистре плейсхолдера в самой строке перевода, `strtr` подставит нужный вариант в зависимости от того, что реально написано в тексте.

## `setLocale()`/`getLocale()`, `setFallback()`/`getFallback()`

Инстанс-независимые (класс работает только статически) переключатели текущей локали/fallback **на время жизни процесса**, отдельные от `config('locale')`/`config('fallback')`: `setLocale()` пишет в `self::$locale`, `getLocale()` возвращает его, если задан, иначе резолвит `config('locale')` через `resolveLocaleValue()` (поддержка `callable`/`\Closure`-значения конфига). Это осознанное разделение — `setConfig(['locale' => ...])` меняет глобальный дефолт для всех последующих (в том числе будущих) запросов/процессов, тогда как `setLocale()` — точечное решение "для этого запроса", которое не переживает следующий `setConfig()` (тот сбрасывает `self::$sources`/`$excludes`/`$maps`/`$fileData`, но не `self::$locale` — смена локали инструментом рантайма и смена конфигурации класса — независимые оси).
