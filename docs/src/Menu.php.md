# Menu.php
<!-- DOCGEN:END -->

## Что это

`ST_system\Menu` — небольшой класс, рендерящий вложенное (многоуровневое) HTML-меню из массива-описания структуры (или PHP-файла, который такой массив возвращает). В отличие от большинства классов фреймворка, это обычный (не `final`) класс с созданием экземпляра через `new Menu($params)`, без статических хелперов и без использования `Traits\HasConfig`/`Traits\HasInstance`.

Класс рассчитан на рекурсивные структуры вида «секция → элементы → вложенные секции», типичные для навигационных меню сайта, и позволяет полностью настроить HTML-обёртку каждого уровня вложенности (открывающий/закрывающий тег списка и шаблон отдельного пункта) — как статическими строками, так и функциями-рендерерами.

### Формат входных данных меню (`$PARAMS['menu']`)

Ожидается ассоциативный массив узла меню со следующими ключами:

- `'FIELDS'` — произвольные данные узла (например, `NAME` — отображаемое имя, `TYPE` — тип узла: `'SECTION'`, `'ITEM'` и т.п.);
- `'PROPERTIES'` — произвольные дополнительные данные узла, передаваемые в рендер-функции наравне с `FIELDS`;
- `'ITEMS'` — (только для узлов-контейнеров/секций) массив дочерних узлов той же структуры. Если у узла есть `'ITEMS'`, он рендерится рекурсивно как вложенный список; если `'ITEMS'` нет — узел считается листом (конечным пунктом меню).

Пример структуры:

```php
$menuData = [
    'FIELDS'     => ['NAME' => 'root'],
    'PROPERTIES' => [],
    'ITEMS' => [
        [
            'FIELDS'     => ['NAME' => 'Главная', 'TYPE' => 'ITEM'],
            'PROPERTIES' => ['href' => '/'],
        ],
        [
            'FIELDS'     => ['NAME' => 'Каталог', 'TYPE' => 'SECTION'],
            'PROPERTIES' => [],
            'ITEMS' => [
                [
                    'FIELDS'     => ['NAME' => 'Товары', 'TYPE' => 'ITEM'],
                    'PROPERTIES' => ['href' => '/catalog/products'],
                ],
            ],
        ],
    ],
];
```

## Свойства

- `public $menu` — итоговый (нормализованный) массив меню, использованный при рендере; заполняется в конструкторе.
- Приватные: `self::$DEFAULT` (статичный набор HTML-заглушек по умолчанию — `OPEN`/`ITEM`/`CLOSE`), `$params` (настройки экземпляра: `render_empty`, `render_rules`).

## Публичные методы

### `__construct($PARAMS)`

Создаёт объект меню. `$PARAMS` — ассоциативный массив:

- `'menu'` (**обязательный**) — либо готовый массив меню (см. формат выше), либо строка — путь к PHP-файлу, который при `require` вернёт такой массив. Если ключ отсутствует — бросается `\Exception("Не передано меню!")`. Если передана строка-путь и файл не существует — `\Exception("Передана путь к файлу с меню, но файл не найден!")`. Если после подгрузки (или изначально) значение — не массив, бросается `\Exception('Меню не оказалось массивом!')`.
- `'render_empty'` (bool, по умолчанию `false`) — рендерить ли узлы с `FIELDS['TYPE'] === 'SECTION'`, у которых нет дочерних `ITEMS` (см. `render()` ниже).
- `'render_rules'` (array, по умолчанию `[]`) — правила рендера по умолчанию, используемые, если явные правила не переданы в `render()` (см. ниже).

Любые не перечисленные в `$params` (по умолчанию: `render_empty`, `render_rules`) ключи `$PARAMS` игнорируются — конструктор копирует в `$this->params` только известные ключи, если они присутствуют во входном массиве.

```php
use ST_system\Menu;

// Из готового массива
$menu = new Menu(['menu' => $menuData]);

// Из PHP-файла, возвращающего массив меню
$menu = new Menu(['menu' => __DIR__ . '/config/menu.php']);

// С кастомными параметрами рендера по умолчанию
$menu = new Menu([
    'menu'         => $menuData,
    'render_empty' => true,
    'render_rules' => [
        'default' => [
            'OPEN'  => '<ul class="menu">',
            'ITEM'  => fn($FIELDS, $PROPERTIES) => "<li><a href=\"{$PROPERTIES['href']}\">{$FIELDS['NAME']}</a></li>",
            'CLOSE' => '</ul>',
        ],
    ],
]);
```

---

### `render($render_rules = null): string` (реализовано через `__call`)

У класса нет объявленного метода `render()` — вызов `$menu->render(...)` перехватывается магическим `__call($name, $arguments)`, который для `$name === 'render'` запускает приватный рекурсивный рендерер, начиная с корня `$this->menu` (`ITEMS`/`FIELDS`/`PROPERTIES`) и глубины `0`. Любое другое имя метода приводит к `\BadMethodCallException("Метод {$name}() не существует.")`.

Если аргумент `$render_rules` не передан (или передан «пустой» — `empty()`), используются правила из конструктора (`$this->params['render_rules']`).

**Формат `$render_rules`** — ассоциативный массив, где ключ — это либо номер глубины вложенности (`0`, `1`, `2`, ...), либо `'default'` (используется, если для конкретной глубины отдельного правила нет), а значение — массив с тремя ключами:

- `'OPEN'` — HTML, открывающий список на этом уровне: строка **или** `callable(array $FIELDS, array $PROPERTIES): string` (вызывается с данными узла-родителя этого уровня);
- `'ITEM'` — как рендерить один листовой пункт: строка **или** `callable(array $FIELDS, array $PROPERTIES): string` (вызывается с данными конкретного пункта);
- `'CLOSE'` — HTML, закрывающий список (аналогично `'OPEN'`, тоже строка или callable).

Порядок выбора правил для конкретной глубины `$depth` (внутри приватного `render()`): `$render_rules[$depth]` → `$render_rules['default']` → `$render_rules[0]` → встроенный дефолт (`<ul>` / `<li>{FIELDS['NAME']}</li>` / `</ul>`).

Узел с ключом `'ITEMS'` всегда рендерится рекурсивно как вложенный список (глубина `+1`), независимо от `FIELDS['TYPE']`. Узел без `'ITEMS'` (лист) рендерится через `'ITEM'`, только если:
- `render_empty` включён (`true`), **или**
- `FIELDS['TYPE']` не равен `'SECTION'`, **или**
- `FIELDS['TYPE']` равен `'ITEM'`.

(На практике это означает: обычные пункты (`TYPE !== 'SECTION'`, включая явный `'ITEM'`) рендерятся всегда, а «пустые» секции без вложенных `ITEMS` — только если явно включён `render_empty`.)

```php
$html = $menu->render([
    0 => [
        'OPEN'  => '<ul class="top-level">',
        'ITEM'  => fn($FIELDS, $PROPERTIES) => "<li>{$FIELDS['NAME']}</li>",
        'CLOSE' => '</ul>',
    ],
    'default' => [
        'OPEN'  => '<ul class="submenu">',
        'ITEM'  => fn($FIELDS, $PROPERTIES) => "<li><a href=\"{$PROPERTIES['href']}\">{$FIELDS['NAME']}</a></li>",
        'CLOSE' => '</ul>',
    ],
]);

echo $html;

// Без аргументов — используются render_rules, переданные в конструктор
echo $menu->render();
```
