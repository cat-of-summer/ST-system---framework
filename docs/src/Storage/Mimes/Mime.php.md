# Mime.php
<!-- DOCGEN:END -->

## Назначение

`Mime` — базовый класс всех MIME-обработчиков в `ST_system\Storage\Mimes`. Он не привязан к конкретному типу контента и задаёт минимальный контракт, который переопределяют наследники (`CssMime`, `JsonMime`, `ImageMime` и т. д.).

Обработчик MIME не создаётся напрямую пользовательским кодом — его подбирает и инстанциирует `Resource`/`File` по фактическому MIME-типу файла (см. `Resource::resolveMimeService()` и таблицу `mimes.services` в конфиге `Resource`). Сам объект `Mime` хранит ссылку на владеющий его `Resource` в защищённом свойстве `$file` и получает её через конструктор.

## Контракт

- `get($data)` — преобразование сырого содержимого (`string`) в удобное для PHP представление (например, `JsonMime` декодирует JSON в массив). Базовая реализация — тождественная (возвращает `$data` как есть).
- `set($data, int &$flags = 0)` — обратное преобразование перед записью содержимого (например, `JsonMime` кодирует массив в JSON-строку). Базовая реализация — тождественная.
- `purge(bool $storage = true)` — сброс любого внутреннего/файлового кеша обработчика. Базовая реализация сбрасывает `$this->cache`, если оно определено у наследника.
- `toHTML(array $config = []): string` — рендер содержимого в виде HTML-фрагмента (`<link>`, `<script>`, `<img>` и т. п. — в зависимости от типа). Базовая реализация возвращает пустую строку.
- `__init(): void` — protected-хук, вызывается из конструктора после присвоения `$this->file`; наследники переопределяют его, чтобы проинициализировать собственные зависимости (например, `CacheManager`), не трогая финальный конструктор.
- `getAttrString(array $attrs): string` — статический хелпер, собирающий строку HTML-атрибутов из ассоциативного массива (`true` → атрибут без значения, `false`/`null` → атрибут опускается, остальное — экранируется через `htmlspecialchars`). Используется наследниками при генерации тегов в `toHTML()`/`toImg()`/аналогах.

Конструктор `__construct(Resource $file)` объявлен `final` — наследники не переопределяют его, а используют `__init()` для собственной инициализации.

## Как писать новый обработчик

1. Создать класс в `ST_system\Storage\Mimes`, унаследовав его от `Mime`.
2. При необходимости собственного состояния (кеш, конфиг и т. п.) — переопределить `protected function __init(): void`, а не конструктор.
3. Переопределить `get()`/`set()`, если MIME требует преобразования данных при чтении/записи.
4. Переопределить `toHTML()`, если для этого типа контента нужен HTML-рендер (`<link>`, `<script>`, `<img>`, …).
5. При необходимости общих поведений подключить готовые трейты из `Storage\Mimes\Traits` — `Minifiable` (минификация с кешированием), `Combinable` (объединение нескольких файлов в один с кешированием) или `Extractable` (разбор DOM-контента по xpath-схеме для HTML/XML).
6. Зарегистрировать класс в `Resource::config('mimes.services')`, сопоставив ему MIME-тип (или префикс типа, например `'image/'`).

## Использование через File/Resource

Код-потребитель никогда не работает с `Mime`-объектом напрямую — методы вызываются на `File`/`Resource`, а `Resource::__call()` прозрачно проксирует их в `$this->mime()`, кешируя результат по имени метода и хешу аргументов:

```php
$file->get();            // -> Mime::get($file->getRaw())
$file->set($data);       // -> Mime::set($data)
$file->toHTML();         // -> Mime::toHTML([])
$file->purge();          // сбрасывает как атрибуты Resource, так и кеш Mime
```

Методы, специфичные для конкретного обработчика (`minify()`, `combine()`, `extract()`, `convert()`, `bySprite()` и т. д.), вызываются на `$file` точно так же — через тот же механизм `__call`.
