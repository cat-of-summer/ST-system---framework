# SvgMime.php
<!-- DOCGEN:END -->

`ST_system\Storage\Mimes\SvgMime` — MIME-обработчик для контента типа `image/svg+xml`. Наследует `Mime` (см. `docs/src/Storage/Mimes/Mime.php.md`) и подключает трейты `Minifiable` и `Combinable` (см. их документацию в `docs/src/Storage/Mimes/Traits/`). Самый функционально насыщенный обработчик в семействе `Mimes` — умеет инлайнить SVG прямо в HTML, собирать спрайты из нескольких файлов и переписывать `id`, чтобы несколько инлайновых SVG на одной странице не конфликтовали.

## Публичные методы

- **`toImg(array $config = []): string`** — рендерит SVG как обычный `<img src="...">` (не инлайновый SVG): `alt` по умолчанию — имя файла, дополнительно подмешиваются атрибуты исходного корневого `<svg>` (`getSourceAttributes()`), если не передан `clear: true` в `$config`. Явно переданные в `$config` атрибуты имеют приоритет.
- **`toHTML(array $config = []): string`** — рендерит `<svg><use href="путь_к_файлу"></use></svg>` — ссылку на файл без инлайна содержимого. `clear: true` в `$config` отключает подмешивание исходных атрибутов корня.
- **`extract(array $config = []): string`** — **инлайнит** содержимое SVG-файла целиком: парсит через DOM (либо через regex-фоллбэк, если `DOMDocument` недоступен), переписывает **все** `id`-атрибуты, добавляя суффикс `_N` (статический счётчик `$counter`, инкрементируется на каждый вызов), и синхронно правит все ссылки вида `url(#старый_id)` на новые id — это защита от коллизий `id`/`url(#...)`, когда на одной странице инлайнится несколько копий одного и того же SVG. `clear: true` убирает атрибуты корневого `<svg>`; остальные ключи `$config` становятся атрибутами корневого `<svg>` (заменяя существующие с тем же именем).
- **`bySprite(string $id, array $config = []): string`** — рендерит ссылку на элемент спрайта: `<svg ...><use xlink:href="путь_к_файлу#id"></use></svg>`, где `$config` — произвольные атрибуты корневого `<svg>`.
- **`extractSprite(string $id, array $config = []): string`** — **извлекает и инлайнит** один `<symbol id="...">` из спрайт-файла как отдельный `<svg>` (переносит `viewBox` символа, если есть, плюс атрибуты из `$config`). Возвращает пустую строку, если символ с таким `id` не найден.

## Combine и Minify (через трейты)

- **`combine($files, $config)`** (из `Combinable`) для SVG собирает **спрайт**: каждый входной файл превращается в `<symbol id="...">` (беря `id` из корневого `<svg>` файла, а если файл сам уже содержит `<symbol>`-элементы — копируя их как есть) внутри одного `<svg style="display:none">`. Коллизии `id` между файлами разрешаются суффиксом `_2`, `_3`, ... `viewBox` каждого символа сохраняется.
- **`minify($content, $config)`** (из `Minifiable`) для SVG — убирает XML-комментарии, схлопывает пробелы между тегами и повторяющиеся пробелы.

Оба метода реализованы через приватные хуки `__combine()`/`__minify()`/`__combineExtension()` (расширение результата комбинирования — всегда `svg`), которые вызывают `Combinable`/`Minifiable` из своих публичных методов (см. их документацию).

## Пример использования (через `File`/`Assets`)

```php
$icon = \ST_system\Storage\File::make('icons/user.svg');

echo $icon->toImg(['class' => 'icon']);      // <img src="..." class="icon" ... />
echo $icon->extract(['class' => 'icon']);    // инлайновый <svg class="icon">...</svg>, id переписаны
echo $icon->bySprite('user', ['class' => 'icon']); // <svg class="icon"><use xlink:href=".../sprite.svg#user"></use></svg>
```

На практике чаще используется через `Assets::svg()`/`Assets::sprite()` (см. `docs/src/Assets.php.md`), которые оборачивают эти же вызовы.
