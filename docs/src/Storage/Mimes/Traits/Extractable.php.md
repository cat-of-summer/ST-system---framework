# Extractable.php
<!-- DOCGEN:END -->

## Назначение

`trait Extractable` — контракт и движок извлечения структурированных данных из DOM-ориентированного тела (HTML/XML). Подмешивается в `HtmlMime` и `XmlMime`.

Реализующий mime-класс задаёт только `loadDom()` (у `HtmlMime` — через `loadHTML`, у `XmlMime` — через `loadXML`); сам трейт предоставляет общий движок: ленивое построение DOM и XPath (`getDom()`/`getXPath()`) и рекурсивную выборку данных по xpath-схеме (`extract($schema, $data)`).

Трейт работает как с in-memory ресурсом (например, ответом `WebClient`), так и с обычным `File` — тело всегда берётся через `$this->file->getRaw()`.

### Кеширование результата `extract()`

Результат извлечения кешируется по составному ключу:
- идентификатор ресурса — `$this->file->getId()` (у `File` это pathname, у `WebClient` — ключ запроса);
- плюс хеш самой схемы и переданных данных (`Main::hash([$schema, $data])`).

Инвалидация кеша:
- для файлов на диске — по `mtime` исходного файла;
- для in-memory ресурсов (mtime недоступен/нулевой) — по `Main::hash()` содержимого тела: при том же `id`, но изменившемся содержимом, кеш пересчитывается.

Если у ресурса нет идентификатора (анонимный ресурс, `getId() === ''`), кеширование не выполняется — схема применяется каждый раз заново.

## Публичные методы

### `getDom(): \DOMDocument`

Возвращает построенный DOM-документ, лениво создавая его через `loadDom((string)$this->file->getRaw())` при первом обращении и кешируя внутри объекта (в приватном свойстве `$dom`).

### `getXPath(): \DOMXPath`

Возвращает `\DOMXPath`, построенный поверх `getDom()`; так же лениво создаётся один раз и переиспользуется.

### `extract(array $schema, array $data = []): array`

Выполняет выборку данных из DOM по xpath-схеме, с прозрачным кешированием результата (см. выше).

Параметры:
- `$schema` — схема извлечения (описание формата — ниже).
- `$data` — произвольные дополнительные данные, которые прокидываются в callable-экстракторы схемы (например, контекст запроса).

## Формат xpath-схемы

Схема — ассоциативный массив, где каждый ключ (кроме служебных ключей, начинающихся с `@`, — они пропускаются) описывает одно извлекаемое поле. Значение — либо строка-селектор, либо массив с ключами:

- `@xpath` (string, обязателен) — xpath-выражение.
  - Если строка начинается с `~` — это **глобальный** запрос: выполняется относительно всего документа (`~` отбрасывается), независимо от текущего контекста вложенности.
  - Иначе, если текущий контекст — не сам `\DOMDocument` (то есть мы уже внутри вложенной выборки), селектор приводится к относительному виду (гарантированно начинается с `.`), чтобы xpath-запрос выполнялся относительно текущего узла, а не всего документа.
- `@extract` (callable|array|null, необязателен) — как преобразовать найденные узлы:
  - не задан (`null`) — берётся текстовое содержимое узла (`nodeValue`), из которого убираются неразрывные пробелы (` `) и переводы строк, а результат обрезается (`trim`);
  - `callable` — вызывается как `$extract($node, $data)` для каждого найденного узла (либо один раз для первого узла, если `@array` = false); должен вернуть готовое значение поля;
  - `array` — вложенная xpath-схема: для каждого найденного узла рекурсивно применяется `applySchema()` с этим узлом как новым контекстом (позволяет строить вложенные структуры/списки объектов).
- `@array` (bool, по умолчанию `true`) — если `true`, результат поля — массив значений по всем найденным узлам; если `false` — берётся только первый найденный узел (или `null`, если ничего не найдено).

Строковое значение поля схемы (`'field' => 'selector'`) — это сокращение для `['@xpath' => 'selector']` (текст узла, массив значений).

Схема может быть вложенной произвольной глубины через `@extract` в виде массива — так строятся списки объектов и вложенные структуры.

### Прочее

- `protected function purgeDom(): void` — сбрасывает закешированные `$dom`/`$xpath`, чтобы следующий вызов `getDom()`/`getXPath()` перечитал тело заново.
- `abstract protected function loadDom(string $content): \DOMDocument` — точка расширения: реализующий класс решает, как парсить строку в DOM (`loadHTML` vs `loadXML`).

## Пример использования

```php
// $html — экземпляр HtmlMime
$data = $html->extract([
    'title'    => '//title',
    'headings' => ['@xpath' => '//h2', '@array' => true],
    'links'    => [
        '@xpath'   => '//a',
        '@array'   => true,
        '@extract' => fn(\DOMNode $a) => $a->attributes->getNamedItem('href')?->nodeValue,
    ],
    'article'  => [
        '@xpath'   => '//article',
        '@array'   => false,
        '@extract' => [
            'body'  => '.',
            'image' => ['@xpath' => './/img', '@array' => false,
                        '@extract' => fn($img) => $img?->getAttribute('src')],
        ],
    ],
]);
```
