# Minifiable.php
<!-- DOCGEN:END -->

## Назначение

`trait Minifiable` добавляет минификацию файла-ассета с кешированием результата на диске. Подмешивается в `CssMime`, `JsMime`, `SvgMime`, `FontMime`.

Сам трейт не знает, как минифицировать конкретный формат, — он организует получение исходного файла (в т.ч. по URI), проверку/инвалидацию кеша по `mtime` и запись результата. Конкретный алгоритм минификации (`__minify`) реализует класс, который подмешивает трейт.

Трейт использует `use HasConfig;` — конфигурация по умолчанию задаёт `cache_dir` через `File::config('cache.dir')`.

## Публичные методы

### `minify(array $config = []): File`

Минифицирует файл и возвращает результирующий `File` (уже готовый закешированный минифицированный файл, либо исходный, если он уже минифицирован).

Параметры:
- `$config` — опции:
  - `force` (bool) — принудительно пересобрать минифицированную версию, даже если кеш свежий.

Поведение:
1. Если текущий файл — `is_uri`, сначала выполняется `fetch()` для получения локального экземпляра.
2. Если файл не существует — бросает `\InvalidArgumentException` с сообщением `"File not found: {путь}"`.
3. Если файл уже помечен как минифицированный (`is_minified`) — возвращается как есть, без повторной обработки.
4. Кеш-файл определяется как `{basename}.min.{extension}`, кеш организован через `CacheManager` (см. `__init()`), ключ кеша строится от пути оригинала (`getOriginal(true)->getPathname()`).
5. Пересборка происходит, если передан `force => true`, либо кеш-файла ещё нет на диске, либо сохранённая в метаданных кеша отметка `stamp` не совпадает с текущим `mtime` файла (инвалидация по mtime-штампу, а не по прямому сравнению времени модификации исходника).
6. При пересборке в кеш записывается результат `__minify($raw, $config)` (или исходное содержимое `$raw`, если `__minify` вернул пустое значение) вместе с новым `stamp` в метаданных.
7. Возвращает результирующий объект через `$instance->make($cache->file)`.

## Расширяемость

Классы, использующие трейт, обязаны реализовать:
- `abstract protected function __minify(string $raw, array $config)` — собственно алгоритм минификации содержимого; должен вернуть минифицированную строку (или falsy — тогда используется исходное содержимое без изменений).

Внутренняя инициализация кеша (`protected function __init(): void`) вызывается инфраструктурой класса-владельца и создаёт `$this->cache` — `CacheManager` с драйвером `filesystem`, директорией из конфига (`cache_dir`) и именем файла — исходным именем файла.

## Пример использования

```php
// $js — экземпляр JsMime
$minified = $js->minify(['force' => false]);

echo $minified->getPathname(); // путь к минифицированному и закешированному .js
```
