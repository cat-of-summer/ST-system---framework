# Minifiable.php

## 1. Концепция

Трейт для MIME-сервисов, которые умеют минифицировать контент (CSS, JS, SVG, шрифты). Хранит минифицированный файл в кэше с TTL=-1 (навсегда). Повторный вызов `minify()` возвращает кэш без пересчёта, если исходный файл не изменился.

Актуальность определяется **штампом**: в метаданных минифицированного файла лежит `stamp` — `mtime` исходника на момент сборки. Разошёлся с текущим `mtime` — файл пересобирается.

```php
// Используется через File::make() -> MIME-сервис
$css = File::make('~/assets/app.css');
$minified = $css->minify();       // CssMime::minify()
$minified = $css->minify(['force' => true]); // принудительная перегенерация
```

> **Про `mtime`.** У него секундная гранулярность, и он читается из stat-кэша PHP. В долгоживущем процессе, где исходник правит другой процесс, `mtime` сам не обновится — вызовите `$file->purge(false)` перед `minify()`, иначе пересборка не сработает.

Трейт объявляет `protected Cache $cache`, поэтому `Mime::purge()` подхватывает его автоматически, без переопределения.

## 2. Публичные методы

### `minify(array $config = []): File`
Возвращает `File` на минифицированный файл (`*.min.ext`). Пересобирает, если файла нет, передан `force`, либо `stamp` в метаданных кэша не равен `mtime` исходника.

### `abstract __minify(string $content, array $config): string`
Должен быть реализован в использующем классе. Принимает сырой контент, возвращает минифицированный.
