`Resource` — базовый класс-носитель контента и mime-сервиса, не привязанный к диску или сети:
хранит тело в памяти, резолвит mime по расширению/content-type и проксирует вызовы либо к
`\SplFileInfo` (построенному из «имени» ресурса), либо к сервису-обработчику из `Mimes/`.
Операции, требующие реального носителя (`fetch`, `find`), в базовом классе бросают исключение —
они реализованы только в наследнике.

`File extends Resource` — практический, материализованный ресурс: локальный путь файловой
системы либо удалённый URI, обрабатываемые единым интерфейсом. Для URI добавляется fetch с
локальным файловым кешем (через `HTTP\WebClient`), независимый кеш HTTP-метаданных (HEAD-запросы,
свежесть), per-host рейт-лимитинг и glob-based поиск множества файлов (`find`). Для локальных
путей большинство операций — прямые обёртки над файловой системой. Именно `File` — рабочая
лошадка, используемая `Assets.php`, `Loader.php` и другими компонентами для доступа к любым
файлам, локальным или удалённым.

Поддиректория [`Mimes/`](Mimes/) содержит семейство mime-обработчиков (CSS/JS/HTML/Image/JSON/
SVG/XML/Font/PlainText), которые `Resource`/`File` резолвят и вызывают для типизированной работы
с контентом (декодирование/кодирование, доступ к специфичным для формата операциям).
