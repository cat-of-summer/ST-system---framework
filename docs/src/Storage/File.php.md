# File.php
<!-- DOCGEN:END -->

`final class File extends Resource` — представляет либо локальный путь файловой системы, либо
удалённый URI, прозрачно объединяя доступ к файлам, remote fetch с локальным кешем (через
`HTTP\WebClient`), метаданные (`getMeta`/HEAD-запросы), glob-based поиск множества файлов (`find`)
и утилиты дискового пространства. Это один из самых нагруженных классов проекта: почти каждый его
метод форкается по внутреннему флагу `is_uri` — локальный путь и удалённый URL обрабатываются
разными ветками кода, но наружу выглядят одинаково (один и тот же публичный интерфейс).

Зависимости: `Main` (резолв путей, форматирование байт), `HTTP\WebClient` (remote GET/HEAD),
`Cache\CacheManager` (кеш фетченого контента + кеш HTTP-метаданных), родитель `Storage\Resource`
(SplFileInfo-проксирование, mime-резолв/диспатч, in-memory буфер). Используется повсеместно
`Assets.php`, `Loader.php` и косвенно `Access.php`.

## Конфигурация

Мёржится с mime-конфигом родителя (`Resource::getDefaultConfig()`):

- `cache.dir` — директория файлового кеша, по умолчанию `<CacheManager default dir>/File`.
- `cache.ttl` — TTL кеша по умолчанию (наследуется от `CacheManager::config('default.ttl')`).
- `request.headers` (`[]`), `request.follow_redirects` (`true`), `request.delay` (`0` мс),
  `request.connect_timeout` (`10` с), `request.timeout` (`300` с), `request.max_attempts` (`15`).
- `find.max_files` (`50`), `find.sym_links` (`false`), `find.recursive` (`true`),
  `find.hidden_files` (`false`).

## Конструирование

Конструктор `__construct(string $path, array $options = [], $original = null)` объявлен
`protected` — напрямую через `new File(...)` инстанс не создать, только через фабрику `make()`
(унаследованную от `Resource::make`) или сахарные статические методы.

Детект режима происходит один раз, в момент создания: `filter_var($path, FILTER_VALIDATE_URL)`.
Результат кладётся в `$options['is_uri']` и после этого доступен как атрибут `is_uri`
(геттер объявлен в `Resource::getIsUriAttribute()`, сеттер — no-op, атрибут неизменяем).

Если путь не URI, он резолвится через `Main::preparePath($path, $base)`, где `$base`:
- директория оригинала (`$original->getDirectory()`), если `$original` — не-URI `File`
  (так строится цепочка относительного резолва: дочерний файл ищется относительно родителя);
- иначе числовая «глубина вызывающего кадра» `2` — путь резолвится относительно кода, который
  вызвал фабрику (обычный механизм `Main::preparePath` для caller-relative путей).

Если путь — URI, он используется как есть (без резолва).

## Статический и инстанс-сахар (`__callStatic` / `__call`)

`__callStatic` реализует сахар для вызовов вида `File::method(...)` без явного `make()`:

- `File::make($path, $options)` → `static::make($path, $options)`.
- `File::fetch($path, $force = false, $options = [])` → `File::make($path, $options)->fetch($force)`.
- `File::find($input, $config = [])` → `static::find($input, $config)`.
- `File::exists($path)` → `File::make($path)->exists()`.
- Любое другое имя бросает `\Exception("Method {$name} not found")`.

`__call` — инстанс-версия:

- `fetch`/`exists` — прямой проброс к одноимённым приватным методам.
- `make($path, $options = [])` — строит **дочерний** `File`, передавая `$this` как `$original`
  (третий аргумент конструктора). Это и есть механизм цепочки относительного резолва: `$file
  ->make('sibling.txt')` ищет `sibling.txt` рядом с `$file`.
- `find($pattern, $config = [])` — если текущий файл сам URI, делегирует статическому `find()`
  без изменений. Иначе сначала резолвит `$this->fetch()->getDirectory()` (если `$this` —
  remote-файл, это неявно фетчит его в локальный кеш и берёт директорию кеш-файла), затем
  каждый элемент входного массива паттернов:
  - объекты `File` и URL-строки (`FILTER_VALIDATE_URL`) оставляет как есть;
  - строки, начинающиеся с `/` или `~`, тоже оставляет как есть (уже абсолютные);
  - остальные («голые» относительные имена/паттерны) склеивает с резолвленной директорией:
    `$dir.'/'.ltrim($i, '/')`.

  Результат передаётся в статический `find()`.
- `getType()` — если `is_uri`, коротко возвращает `'uri'` без обращения к родительскому
  SplFileInfo-проксированию; иначе падает на `parent::__call('getType', ...)`.
- Всё остальное уходит в `parent::__call()` (SplFileInfo-методы, затем mime-сервис).

## Кеш контента (`cache()`)

Приватный мемоизированный метод `cache(): CacheManager` строит filesystem-кеш-хендл, ключуемый по
`getPathname()` (для URI — сам URL, для локального файла — путь).

Для URI имя кеш-файла дополнительно нормализуется: query-string отрезается от базового имени,
хешируется (`md5($query)`) и добавляется как суффикс перед расширением. Например,
`img.jpg?w=200` превращается в `img_ab12cd34.jpg`. Благодаря этому разные query-варианты одного
URL (те же `img.jpg`, но с разными `?w=...`) не коллизируют в одном кеш-файле — у каждого набора
query-параметров свой файл на диске.

## `find()` — glob-резолвер множества файлов

Приватный статический метод, вызываемый только изнутри (`__callStatic`/`__call`). Центральный
механизм поиска файлов по одному или нескольким паттернам сразу.

Конфиг мёржится с `static::config('find')`; `extension`, если задан не массивом, оборачивается в
массив.

Вход (`$input`) может быть одиночным элементом или массивом; каждый элемент обрабатывается
независимо (через `array_walk`) и может быть:

1. **Существующим `File`** — включается напрямую (дедуп по `getPathname()` через локальную карту
   `$seen`, общую на весь вызов `find()`).
2. **URL-строкой** (`FILTER_VALIDATE_URL`) — оборачивается через `self::make($item)` и включается
   без похода на диск (remote-ресурсы не проверяются на существование заранее).
3. **Путём или glob-паттерном файловой системы** — обрабатывается по одной из двух веток:

   - **Без глоб-метасимволов** (regex `/[\(\[\{\|\?\+\*]/` не совпал): если путь существует и это
     директория — итерируется (`FilesystemIterator`/`RecursiveIteratorIterator`, в зависимости от
     `recursive`; уважает `sym_links` через `FOLLOW_SYMLINKS`), фильтруется по `extension`,
     собирается не больше `max_files` файлов. Если это обычный файл — включается напрямую.
   - **С глоб-метасимволами**: вычисляется «стартовая директория» — самая глубокая конкретная
     (не содержащая глоб-символов) часть пути перед первым метасимволом. Если этой директории нет
     или она нечитаема — **бросает** `\InvalidArgumentException`. Из оставшейся части пути строится
     PCRE-регулярка (delimiter выбирается автоматически как первый символ из набора
     `# ~ % @ ! $ \` ; |`, не встречающийся в самом паттерне, чтобы не коллизировать с
     glob-синтаксисом). Каждый файл, найденный итератором по стартовой директории, матчится против
     этой регулярки; сегменты пути, начинающиеся с точки, пропускаются, если не включён
     `hidden_files`.

   Если после обеих веток результатов нет:
   - при `fallback === 'make'` (**только для не-glob путей**) элемент всё равно оборачивается в
     `File::make()` — отсутствующий файл трактуется как ленивая ссылка, а не как ошибка; именно
     этот режим использует `Assets.php` (`find($args[0] ?? '', ['fallback' => 'make'])`);
   - при `fallback === 'throw'` — бросает `\InvalidArgumentException("File not found: {$item}")`;
   - иначе элемент молча пропускается.

Внутри используется статик-локальный `$input_cache` (мемоизация `Main::preparePath` для
повторяющихся входных строк в рамках процесса) и карта `$seen` для дедупа результатов в рамках
одного вызова `find()`.

## Вклад в кеш View (просмотренные каталоги)

Раскрывая директорию или glob, `find()` отмечает **просмотренные каталоги**: состав такого
каталога — часть входных данных, и появление в нём нового файла должно инвалидировать кеш
потребителя. Иначе новый файл в `js/lib` (см. `Assets`) остался бы незамеченным, пока не изменится
mtime шаблона.

`File` — сам вкладчик в кеш [`View`](../View.php.md#вкладчики-в-кеш-через-события): публично
торчит только **`static registerViewEvents(): void`** (идемпотентна), которую `View` находит по
имени и зовёт из конфига `contributors`. Внутри она подписывается на:

- `cache_open` — кладёт новый кадр в стек просмотренных каталогов;
- `cache_close` — снимает кадр и подмешивает его каталоги в `$deps` границы (инвалидация по mtime:
  `depStamp()` использует `file_exists()`, а не `is_file()`, ровно чтобы ловить mtime каталога).

`find()` пишет в **верхний** кадр стека, поэтому каталог приписывается той границе кеша, во время
рендера которой его просматривали. Вне рендера стек пуст — просмотры не копятся. Так `Assets`
больше не таскает эти данные через публичный метод: `File` ведёт их сам.

## Атрибуты (`attributeMap()`)

Расширяет родительскую карту атрибутов, добавляя magic-геттеры: `real_path`, `directory`, `size`,
`type`, `exists`, `ctime`, `atime`, `is_dir`, `is_file`, `is_readable`, `is_writable`, `is_link`.

## `purge(bool $storage = true): self`

Переопределяет родителя. Для локальных файлов чистит PHP `stat`-кеш (`clearstatcache`). Если
`$storage` — чистит весь кеш файла (`cache()->purge(true)`, включая базовые метаданные), иначе
только instance-level кеш (и только если объект кеша уже был построен, чтобы не создавать его
впустую). Обнуляет in-memory `raw`-буфер (унаследованное свойство `Resource::$raw`), затем
делегирует `parent::purge($storage)`.

## Метаданные (`getMeta` / `setMeta`)

`setMeta(array $meta, bool $append = true): self` — персистит метаданные в собственном
`CacheManager`-файле (`cache()->setMeta(...)`).

`getMeta(bool $force = false): array` — для локальных файлов простой проброс к
`cache()->getMeta()`. Для URI логика гораздо богаче:

- Если не `$force` и метка `headers_cache_expires_in` в текущей мете ещё не истекла
  (`!$this->cache()->isExpired('headers_cache_expires_in')`), возвращается уже закешированная
  мета без сетевого запроса.
- Иначе выполняется реальный HEAD-запрос через `WebClient::create($url, ['method' => 'head', ...
  ])->send()`, используя настроенные для этого файла заголовки/редиректы/таймауты (`verify =>
  false`), с собственным файловым кеш-конфигом (`cache.use = true`).
- При успешном ответе (`errno === 0`) заголовки ответа мёржатся в мету вместе с `http_code`,
  `effective_url`, свежим `headers_cache_expires_in` (через `resolveCacheTtl`) и
  `content_length`.
- Если HEAD вернул код, отличный от 200, но локальный кеш-файл контента физически существует на
  диске — дополнительно продлевается `expires_in` (свежесть уже закешированного контента),
  используя ту же TTL-логику. Смысл: неудачный HEAD-запрос (например, временный сбой сервера) не
  должен аннулировать валидность уже успешно закешированного ранее контента.
- Любое исключение из HEAD-запроса глотается (`catch (\Throwable)`), метод падает на уже
  имеющуюся мету.

## `fetch(bool $force = false): self` — ядро remote fetch + локального кеша

Вызывается только через `__call` (метод `private`). Сначала `purge(false)` — чистит instance-level
кеш-объект и in-memory raw-буфер (не трогая сами файлы на диске).

Если файл не URI — возвращается `$this` без изменений (no-op для локальных путей).

Для URI:

1. Строится GET-запрос через `WebClient::create()`: `verify` выставляется в `true` **только**
   если URL начинается с `https://` (проверка строкового префикса, а не общий конфиг), `exception
   => false` (ошибки не бросаются как исключения, а возвращаются в структуре ответа), `requeue =
   max(0, max_attempts - 1)` (суммарно до `max_attempts` попыток), собственный файловый кеш-конфиг
   с `use = !$force`.
2. Подключается слушатель `on('error', ...)`, аналогичный `IntegrationDriver::attachRetry`:
   помечает `$res['requeue'] = true` при curl-ошибке (`errno !== 0`) или HTTP 5xx.
3. Перед отправкой вызывается `applyHostDelay($url)` — рейт-лимит по хосту.
4. `$client->send()[0]` — берётся первый (и единственный) ответ.

Обработка сбоев:
- Если во время `send()` брошено исключение: если ранее закешированный файл физически существует
  на диске — молча возвращается новый `File`, обёрнутый вокруг этого устаревшего кеш-файла
  (деградация вместо падения); иначе исключение перебрасывается, обёрнутое в `\RuntimeException`.
- Если ответ вернулся без исключения, но неуспешен (`errno !== 0` или `status >= 400`): та же
  логика — фоллбэк на существующий кеш-файл, иначе `\RuntimeException` с деталями HTTP/curl
  ошибки.

При успехе: если ответ не был кеш-хитом (`empty($r['cached'])`) или локальный кеш-файл ещё не
существует — тело ответа пишется во временный файл `{file}.temp`, затем атомарно
переименовывается (`rename()`) на итоговое имя. Это классический write-then-rename паттерн,
защищающий от чтения частично записанного файла параллельным запросом.

Метаданные пересчитываются и **полностью перезаписываются** (`setMeta(..., false)`, не
`append`): `fetched_at`, `cache_ttl`, `expires_in`, `headers_cache_expires_in`, `original_url`,
плюс заголовки ответа, `http_code`, `effective_url`, `content_length`.

Возвращает **новый** инстанс `File`, оборачивающий теперь-локальный кеш-файл, с `$this` в
качестве `$original` — это сохраняет цепочку provenance (через `getOriginal()` всегда можно
добраться до исходного URI-объекта) и относительный резолв для последующих `make()`/`find()`.

## Прочие методы

- `getMtimeAttribute(): int` — `0` для URI (нет локального mtime), иначе `filemtime()`
  (подавленный `@`) или `0`, если это не файл.
- `getSize(string $unit = 'b')` — для URI читает `content-length`/`content_length` из
  `getMeta()` (что может триггернуть HEAD-запрос, если мета устарела); для локальных файлов —
  `info()->getSize()`. Результат форматируется через `Main::formatBytes()`.
- Геттеры атрибутов запроса (`getHeadersAttribute`, `getDelayAttribute`,
  `getFollowRedirectsAttribute`, `getConnectTimeoutAttribute`, `getTimeoutAttribute`,
  `getMaxAttemptsAttribute`) — каждый читает либо явный per-instance оверрайд из
  `$this->attributes[...]` (переданный через `$options` в конструкторе/`make()`), либо
  фоллбэк на `static::config('request.*')`.
- `resolveCacheTtl(array $headers): int` — `max(server-заявленный max-age/s-maxage/Expires,
  static::config('cache.ttl'))`. Заголовки сервера могут только **расширить** TTL сверх
  настроенного минимума, но никогда не уменьшить его ниже пола.
- `applyHostDelay(string $url): void` — простой per-host рейт-лимитер через статическую
  in-process карту `$last_fetch_per_host[host] => microtime`. Если с последнего фетча к этому
  хосту прошло меньше `delay` миллисекунд — досыпает (`usleep`) остаток. Ограничение
  процесс-локальное: не работает между отдельными запросами/воркерами.
- `diskFreeSpace`/`diskTotalSpace(string $path = '~', string $format = 'b')` — статические
  тонкие обёртки над `disk_free_space`/`disk_total_space`, резолвя `~`-относительные пути через
  `Main::preparePath`. Бросают `\RuntimeException`, если системный вызов провалился.
- `purgeAll(): void` — статический метод, чистящий **весь** namespace кеша `File` (все
  закешированные remote-фетчи всех инстансов этого класса), через
  `CacheManager::make('', [...])->purgeBase()`.
- `getMime(): string` — переопределяет родителя. Сначала пробует `parent::getMime()`
  (расширение → таблица `mimes.extensions` или `mime_override`). Если пусто и это URI — читает
  `content-type` из `getMeta()`. Если пусто и у инстанса есть `$original`, который сам URI —
  заимствует **его** content-type (случай, когда `$this` — уже локально-кешированная копия
  remote-оригинала, у которой самой по себе нет заголовков). Иначе — фоллбэк на определение
  по содержимому локального файла: `detectMime()` из трейта [[HasMime.php]]
  (`finfo_file`, затем `mime_content_type`), общего с `HTTP\UploadedFile`.
- `exists(): bool` (`private`) — всегда `false` для URI (существование remote-ресурса
  бессмысленно проверять без фетча), иначе `file_exists()`.
- `getDirectory(int $depth = 0): string` (`public`) — стартовая директория: сам путь, если это
  директория, иначе родительская директория файла; затем поднимается на `$depth` уровней выше
  через повторный `dirname()`.
- `getRaw(bool $force = false)` (`public`) — для локальных файлов обычный
  `file_get_contents($this->getRealPath())`. Для URI: мемоизирует результат в `$this->raw`, если
  не `$force`; иначе триггерит `fetch()` и читает результирующий локальный кеш-файл.
- `putContents($raw, int $flags = 0)` (`public`) — если это URI, сначала фетчит файл
  (`$this->fetch()`) и пишет в **локальный кеш-инстанс**, а не в сам URL (писать напрямую в
  remote-ресурс невозможно). Контент прогоняется через mime-сервис родителя
  (`$instance->mime()->set($raw, $flags)`), целевая директория создаётся при необходимости
  (`@mkdir(..., 0775, true)`, бросает `\RuntimeException` при неудаче), затем
  `file_put_contents`.

## Ключевые "режимы" поведения

- **Прозрачная двойственность URI vs локальный путь**: почти каждый метод форкается по
  `$this->is_uri`, но наружу класс выглядит единообразно — вызывающий код не обязан знать, с чем
  имеет дело.
- **Fetch fallback на устаревший кеш**: сетевые сбои (исключение, статус ≥400, curl errno) не
  приводят к падению, если на диске уже есть ранее успешно закешированный контент — деградация
  плавная, пока есть хоть какая-то кешированная копия.
- **Два независимых слоя кеша для URI**: (1) фетченый **контент**, кешируемый в локальный файл с
  атомарной записью (temp + rename); (2) HTTP **метаданные** (заголовки, свежесть), кешируемые
  независимо от контента через отдельные HEAD-запросы — контент и метаданные могут иметь разное
  время жизни/свежести.
- **Per-host рейт-лимитинг** запросов (`request.delay`), процесс-локальный.
- **Glob-based `find()`** с плавным фоллбэком «сделай всё равно» (`fallback => 'make'`) или
