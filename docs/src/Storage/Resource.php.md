# Resource.php
<!-- DOCGEN:END -->

`class Resource` — базовый (родительский для `File`) носитель контента и mime-сервиса, **не
привязанный к диску или сети**. Можно представлять как «именованный in-memory блоб + mime +
резолвер»: тело хранится в памяти (`$raw`), «имя» ресурса используется для построения
`\SplFileInfo` (откуда берутся `getExtension()`, `getPathname()` и т.д., нужные для резолва mime).

`File` наследует от `Resource` и переопределяет дисковые/сетевые операции (`fetch`, `find`,
`getRaw`, `exists`, `getMime` и др.), добавляя материализацию на диске/по HTTP. Сам `Resource`
для этих операций либо не имеет смысла (нет источника данных, кроме уже переданного тела), либо
явно бросает исключение, сигнализируя, что нужен «материализованный» `File`.

## Конструирование

Конструктор `protected function __construct(array $config = [])` принимает конфиг-массив (а не
путь, как у `File`):

- `name` — «имя» ресурса, используется для построения `SplFileInfo` (по умолчанию `'inline'`).
- `id` — строковый идентификатор контента для производных кешей (у `File` это `pathname`, у
  `WebClient` — ключ запроса).
- `body` — исходное in-memory тело (`$raw`).
- `original` — ссылка на «родительский» ресурс (используется для provenance-цепочек: например,
  локально-кешированная копия remote-файла хранит ссылку на исходный URI-объект).
- `options` — становится `$this->attributes` (см. `HasAttributes`).
- `mime` — если задан, кладётся в `$this->attributes['mime_override']` (принудительный mime,
  минуя резолв по расширению).

Прямой инстанс создаётся через `new Resource(['body' => ..., 'mime' => ...])` или
`Resource::make(...)` / `Resource::__callStatic('make', ...)`. `File` строит свои инстансы через
собственные фабрики (путь/URL), не напрямую через этот конструктор.

## Proxy-диспетчер (`__call`)

Практическая суть класса — в том, как он проксирует вызовы неизвестных методов. Порядок важен:

1. Явные спецслучаи: `exists()` → `$this->raw !== null` (для `Resource` — единственный смысл
   «существования»: есть ли в памяти тело); `fetch()`/`find()` → бросают `\LogicException`
   ("Resource '{name}()' requires a materialized File (disk/HTTP)") — это методы, требующие
   диска/сети, которые есть только у `File`; `getBasename()` без аргументов возвращает базовое
   имя без расширения (`$this->info()->getBasename('.'.$this->getExtension())`).
2. **Методы `\SplFileInfo`** — проверяются по кешированной таблице (`array_flip` списка методов
   `SplFileInfo`, построенной один раз через `static`-переменную) и, если имя совпадает,
   проксируются в `$this->info()->{$name}(...)`. Это обязано идти раньше mime-сервиса, потому что
   резолв mime сам зависит от `getExtension()` / `getPathname()`.
3. Всё остальное считается вызовом mime-сервиса: `$this->mime()->{$name}(...)`, с
   мемоизацией результата в `$this->mime_data` по ключу (имя метода, плюс хеш аргументов
   `Main::hash($args)`, если аргументы есть). Благодаря мемоизации повторный вызов одного и того
   же mime-метода с теми же аргументами не пересчитывается.

## Mime-резолв

`getMime(): string` — если задан `mime_override`, возвращает его. Иначе смотрит расширение файла
(`getExtension()`) в таблице `static::config('mimes.extensions')` (например, `js` →
`application/javascript`, `css` → `text/css`, `svg` → `image/svg+xml` и т.д. — полный список в
`getDefaultConfig()`). Если совпадения нет — возвращает `''`.

`mime(): Mimes\Mime` (мемоизировано в `$this->mime`) — строит сервис-обработчик через
`resolveMimeService($this->getMime(), $this)`: ищет первый паттерн из `mimes.services`
(`text/html` → `HtmlMime`, `application/json` → `JsonMime`, `font/` → `FontMime`, `image/` →
`ImageMime` и т.д. — тоже подстрока через `strpos`), чей ключ входит подстрокой в mime-тип. Если
совпадений нет — возвращает анонимный класс-заглушку (`new class($file) extends Mimes\Mime {}`,
поведение по умолчанию из `Mimes\Mime`).

`setMime(string $mime): self` — принудительно переустанавливает mime, но **только если** текущий
mime-сервис уже анонимный (заглушка), или ресурс — URI, или файл ещё не существует на диске;
иначе (реальный, специфичный mime-сервис на существующем локальном файле) вызов молча
игнорируется — mime уже определён по факту существования файла и не должен перебиваться вручную.
При успешной установке сбрасывается кеш `mime_data` и пересобирается `mime`-сервис.

`getServiceName(): string` — возвращает `'Default'` для анонимного (заглушечного) сервиса, иначе
полное имя класса конкретного mime-сервиса.

## Атрибуты и провенанс

`attributeMap()` — карта magic-атрибутов: `relative_path`, `pathname`, `filename`, `basename`,
`extension`, `path`, `service_name`, `original` (все read-only, второй элемент `true`).

`getOriginal(bool $force = false)` — без `$force` возвращает непосредственного `$original`
(может быть `null`). С `$force = true` идёт по цепочке `original -> original -> ...` до конца и
возвращает самый первый (корневой) ресурс в цепочке — полезно, чтобы, имея локально-кешированную
копию remote-файла, добраться до исходного URI-объекта.

`getRelativePath(string $root = '')` — для URI возвращает сам `pathname` (полный URL) без
изменений; для локальных путей вырезает из `pathname` подготовленный корень
(`Main::preparePath('~'.$root, 1)`), получая путь относительно этого корня.

`getIsUriAttribute()`/`setIsUriAttribute()` — `is_uri` читается из `$this->attributes['is_uri']`
(по умолчанию `false`); сеттер — намеренный no-op (атрибут неизменяем после конструирования).

## Контент: чтение и кодирование

`getRaw(bool $force = false)` — базовая реализация просто возвращает in-memory `$raw`; если тела
нет — бросает `\LogicException("Resource has no content (not materialized)")`. `File`
переопределяет этот метод чтением с диска/по сети.

`get()` / `getContents()` (алиас) — декодируют собственное тело через mime-сервис:
`$this->mime()->get($this->getRaw())`.

`set($data, int &$flags = 0)` — кодирует данные в строку через mime-сервис:
`$this->mime()->set($data, $flags)`.

## Очистка (`purge`)

`purge(bool $storage = true): self` — сбрасывает кеш атрибутов (`purgeAttributes()` из
`HasAttributes`) и `mime_data`; если mime-сервис уже был построен — делегирует ему `purge()`.
Если есть `original` — рекурсивно чистит и его тоже. Возвращает `original`, если он есть, иначе
`$this` (примечательно: возвращаемое значение — не обязательно тот же объект, на котором был
вызван `purge`).

## Практическое значение для `File`

`File` получает от `Resource`:
- SplFileInfo-проксирование (`getPathname`, `getExtension`, `getBasename`, `isDir`, `isFile` и
  т.д.) без необходимости самому оборачивать `\SplFileInfo`;
- резолв и диспетчеризацию mime-сервиса (`Mimes\*`) по расширению/content-type, с ленивой
  инициализацией и мемоизацией вызовов;
- механизм provenance через `original` (используется `File::fetch()` для связи
  URI-инстанса с его локально-кешированной копией);
- каркас атрибутов (`HasAttributes`/`attributeMap`) и конфигурации (`HasConfig`), которые `File`
  расширяет своими собственными полями.

`File` переопределяет всё, что требует реального носителя (диск/сеть): `fetch`, `find`,
`getRaw`, `exists`, `getMime`, `purge`, а также добавляет свои атрибуты (`real_path`, `size`,
`type` и т.д.) поверх базовых.
