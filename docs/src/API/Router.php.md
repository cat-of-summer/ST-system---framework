# Router.php
<!-- DOCGEN:END -->

`class Router` (`ST_system\API\Router`) — роутинг **входящих** HTTP-запросов приложения: сопоставление URL текущего запроса (`$_SERVER['REQUEST_URI']`/`REQUEST_METHOD`) с набором зарегистрированных паттернов и передача результата совпадений во внешний обработчик. Это не имеет отношения к исходящим интеграциям из `API\Drivers\*` (там роутинг эндпоинтов делает `IntegrationDriver::registerMethod()`) — `Router` работает в обратную сторону, разбирая запросы, которые пришли **в** приложение.

## Создание инстанса

```php
$router = new Router([
    'key'           => 'main',                 // ключ для последующего Router::apply_parser('main', ...)
    'url_rules'     => [
        ['/users/{id}', 'user_page'],           // [паттерн, полезная нагрузка, необязательный приоритет]
        ['/users/{id}/posts/{post_id}', 'user_post', 5],
    ],
    'rules_handler' => function($PARSER_PARAMS, $URL_PARAMS, $PAGE_PARAMS) {
        return ['PARSER_PARAMS' => $PARSER_PARAMS, 'URL_PARAMS' => $URL_PARAMS, 'PAGE_PARAMS' => $PAGE_PARAMS];
    },
    'apply_once'    => false,   // если true — учитывается только первое совпадение той же приоритетной группы
    'strict_mode'   => false,   // если true — паттерн якорится от начала (`point`) и до конца строки регулярным выражением
    'point'         => '/',    // префикс пути, используемый только при strict_mode = true
    'methods'       => ['GET', 'POST', 'PUT', 'DELETE', 'PATCH', 'OPTIONS', 'HEAD'], // допустимые HTTP-методы
]);
```

Каждый созданный инстанс автоматически регистрируется в статическом реестре `self::$URL_parsers_list` — под ключом `key`, если он передан, иначе под числовым индексом (`[]`). Если `rules_handler` не задан (или не `callable`), используется дефолтный обработчик, просто возвращающий массив `['PARSER_PARAMS' => ..., 'URL_PARAMS' => ..., 'PAGE_PARAMS' => ...]`.

Параметры `apply_once`/`strict_mode`/`point` приводятся к типу первого (дефолтного) значения того же ключа в `$this->params` (`bool`/`int`/`float`/`string`/`array`/`object`/`null`) — при неподдерживаемом типе дефолта бросается `\Exception`. `methods`, в отличие от остальных, обрабатывается отдельно: если передан массив, из него оставляют только элементы, чьи **ключи** есть среди `self::$methods` (`array_intersect_key`), иначе используются все `self::$methods` целиком.

## `apply(...$PAGE_PARAMS)` — применить правила к текущему запросу

`public function apply(...$PAGE_PARAMS)`:

1. Берёт путь текущего запроса через `parse_url($_SERVER['REQUEST_URI'], PHP_URL_PATH)`.
2. Если `$_SERVER['REQUEST_METHOD']` не входит в разрешённые `methods` — бросает `\Exception("Метод {$_SERVER['REQUEST_METHOD']} не доступен")`.
3. Проходит по всем правилам `url_rules` в порядке регистрации. Каждое правило — это `[паттерн, полезная_нагрузка, приоритет?]`; паттерн компилируется в regex через приватный `get_regexp()` (см. ниже) и сверяется с путём запроса (`preg_match`).
4. При совпадении проверяется условие сброса накопленных результатов: `$max_priority < isset($rule_param[2]) ? (int)$rule_param[2] : 0`. Из-за приоритета операторов в PHP (`<` считается раньше тернарника) это фактически `($max_priority < isset($rule_param[2])) ? (int)$rule_param[2] : 0` — то есть сброс (`PARSER_PARAMS`/`URL_PARAMS`/счётчик обнуляются, `max_priority` становится равным приоритету текущего правила) срабатывает **не более одного раза** за вызов `apply()`: только в момент перехода от `max_priority === 0` к первому встреченному правилу с непустым третьим элементом. После этого `max_priority` становится `>= 1`, и последующие правила — независимо от их приоритета — уже никогда не вызывают повторный сброс, а просто добавляются в текущую накопленную группу. На практике это означает, что механизм приоритетов надёжно работает только для отделения "безприоритетных" правил от одной первой встреченной приоритетной группы, а не как полноценная сортировка по приоритету нескольких групп.
5. Если включён `apply_once` и в текущей группе уже есть хотя бы одно совпадение — остальные совпадения той же группы пропускаются.
6. Для каждого подходящего правила в `PARSER_PARAMS` попадает его полезная нагрузка (`rule_param[1]`), а в `URL_PARAMS` — именованные захваты `{placeholder}` из URL (`array_intersect_key($matches, ...)`).
7. В конце вызывает `rules_handler($PARSER_PARAMS, $URL_PARAMS, $PAGE_PARAMS)` и возвращает его результат.

```php
$result = $router->apply('extra', 'page', 'params');
// $result — то, что вернул rules_handler; по умолчанию
// ['PARSER_PARAMS' => [...], 'URL_PARAMS' => [...], 'PAGE_PARAMS' => ['extra', 'page', 'params']]
```

## `get_regexp($pattern)` — компиляция паттерна в regex (приватный)

Паттерн экранируется через `preg_quote($pattern, '#')`. Если `strict_mode` выключен (по умолчанию), паттерн становится "нестрогим" — с необязательными слэшами по краям (`/?...pattern.../?`), то есть совпадает как часть пути. Если `strict_mode` включён, итоговый regex якорится `^{point}` в начале и `$` в конце — весь путь должен совпасть целиком, начиная с указанной точки `point`. Каждый `{name}` внутри паттерна превращается в именованную группу `(?P<name>[^/]+)`, а имена всех плейсхолдеров собираются в `url_params` — они и используются потом для извлечения `URL_PARAMS` из `preg_match`.

## Статический вход: `Router::apply_parser($key, ...$PAGE_PARAMS)`

`public static function apply_parser($key, ...$PAGE_PARAMS)` — находит зарегистрированный по `key` инстанс `Router` в `self::$URL_parsers_list` и вызывает на нём `apply(...$PAGE_PARAMS)`. Возвращает `null`, если инстанс с таким ключом не зарегистрирован, либо если у него пустой список `url_rules` (то есть роутеру нечего сопоставлять).

```php
new Router(['key' => 'main', 'url_rules' => [['/users/{id}', 'user_page']]]);
// где-то в другом месте приложения:
$result = Router::apply_parser('main');
```

## Доступ к параметрам инстанса: `__get()`

`public function __get($name)` — читает значения из `$this->params` (`apply_once`, `strict_mode`, `point`, `methods`). Обращение к любому другому имени свойства бросает `\Exception("Свойство '{$name}' не существует.")`.

```php
$router->methods;      // ['GET', 'POST', ...]
$router->strict_mode;  // false
```
