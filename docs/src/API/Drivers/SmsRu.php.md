# SmsRu.php
<!-- DOCGEN:END -->

`final class SmsRu extends IntegrationDriver` (`ST_system\API\Drivers\SmsRu`) — драйвер для SMS-шлюза [sms.ru](https://sms.ru) (`https://sms.ru/sms/`). Наследует [`IntegrationDriver`](../IntegrationDriver.php.md) — самый минималистичный из драйверов в этом каталоге: один зарегистрированный метод, авторизация через `api_id` в query-параметрах.

## Создание инстанса

```php
$smsru = SmsRu::create('YOUR_API_ID');
```

Конструктор принимает единственный параметр `string $api_id` — если передана пустая строка, бросается `\InvalidArgumentException('Передан некорректный api_id')`. `api_id` сохраняется в приватное свойство и на событии `call` автоматически подмешивается в параметры **каждого** вызова — указывать его вручную в `call()` не нужно.

## Метод `send` — отправка SMS

```php
$smsru->call('send', [
    'to'   => ['+79991234567', '+79997654321'],
    'msg'  => 'Ваш код: 1234',
    'json' => true,
]);
```

Параметры:

- `to` — обязателен, непустой массив получателей. Валидируется как непустой массив (иначе ошибка `"Не передан список получателей"`), затем из него выбрасываются нестроковые элементы (`array_filter(..., 'is_string')`) с переиндексацией (`array_values`). На этапе `on_prepare` массив склеивается в строку через запятую (`implode(',', ...)`) — именно так `sms.ru` ожидает список получателей в запросе.
- `msg` — обязателен, строка — текст сообщения.
- `json` — `nullable|bool` — просить ли API вернуть ответ в JSON.

```php
$smsru->call('send', ['to' => ['+79991234567'], 'msg' => 'Hello']);
```
