## Описание

Парсеры страниц сайта **Prodoctorov.ru**, оба наследуют `Parsers\DefaultParser` и переиспользуют его конвейер загрузки/извлечения данных:

- **DoctorDetailParser** — карточка врача (`.../vrach/{vrach_id}/`): ФИО, аватар, специальности, стаж, языки, направления лечения, место работы, образование, курсы повышения квалификации, членство в ассоциациях, документы. Поддерживает как одиночный запрос по `vrach_id`, так и пакетный (список `vrach_id` -> список карточек).
- **DoctorsReviewsParser** — страница отзывов о враче (`.../vrach/{vrach_id}/otzivi/`): сводка по врачу (имя, аватар, цена приёма, описание, дата обновления) и список отзывов пациентов (автор, дата, оценка, направление, текст отзыва, ответ клиники и т.д.). Работает только с одним `vrach_id` за вызов (пакетные запросы не поддерживаются).

Оба парсера увеличивают `delay` до 5000 мс по сравнению с базовым `DefaultParser`, чтобы снизить нагрузку на сайт при скрапинге.
