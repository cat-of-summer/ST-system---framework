# Isdayoff.php
<!-- DOCGEN:END -->

`final class Isdayoff extends IntegrationDriver` (`ST_system\API\Drivers\Isdayoff`) — драйвер для [isdayoff.ru](https://isdayoff.ru) — публичного API-календаря рабочих/выходных/праздничных дней для России и ряда других стран СНГ. Наследует [`IntegrationDriver`](../IntegrationDriver.php.md), поэтому всё, что касается общего жизненного цикла (`create()`, события, кеш, `call()`/`callMany()`), устроено там же — здесь описан только специфичный для Isdayoff API.

Базовый URL по умолчанию (`getDefaultConfig()`): `https://isdayoff.ru/api/`.

## Создание инстанса

```php
$isdayoff = Isdayoff::create([
    'pre'       => false,   // считать ли предпраздничные дни сокращёнными (значение по умолчанию для getdata)
    'covid'     => false,   // учитывать ли covid-нерабочие дни (значение по умолчанию для getdata)
    'sd'        => false,   // учитывать ли перенос выходных на другие дни
    'delimeter' => null,    // строка-разделитель ответа, максимум 7 символов
]);
```

Конструктор валидирует переданные параметры (`pre`/`covid`/`sd` — `nullable|bool`, `delimeter` — `null` либо непустая строка не длиннее 7 символов, иначе ошибка `"delimeter должен быть строкой длиной до 7 символов"`), приводит `pre`/`covid`/`sd` к `bool` (по умолчанию `false`) и сохраняет их в `$this->SETTINGS` — они используются как значения по умолчанию для одноимённых параметров метода `getdata`, если те не переданы явно при вызове.

## Метод `getdata` — получить статус дней

```php
$isdayoff->call('getdata', [
    'year'  => 2026,
    'month' => 7,
    // либо диапазон дат:
    // 'date1' => '2026-07-01',
    // 'date2' => '2026-07-31',
    'pre'       => true,        // необязательно — иначе берётся из настроек инстанса
    'covid'     => false,       // необязательно
    'sd'        => false,       // необязательно
    'delimeter' => ',',         // необязательно
]);
```

Параметры:

- `year` — `int`, приводится к 4-значной строке (`sprintf('%04d', ...)`).
- `month` — `int`, приводится к 2-значной строке.
- `day` — `int`, приводится к 2-значной строке.
- `date1` / `date2` — граница диапазона: `null`, `\DateTimeInterface` (форматируется в `Ymd`) или строка, парсируемая через `new \DateTime($v)` и также приводимая к формату `Ymd`. Некорректная дата (не распознанная `\DateTime` или содержащая ошибки парсинга) бросает исключение.
- `pre` / `covid` / `sd` — `nullable|bool`; если не заданы явно при вызове, на этапе `on_prepare` подставляются из `$this->SETTINGS` (значений, переданных при создании драйвера).
- `delimeter` — та же валидация, что и в конструкторе (строка длиной до 7 символов); если не задан явно — берётся из `$this->SETTINGS['delimeter']`.

**Обработка ответа.** На событии `prepare_response` сырой ответ API (одна строка из цифр 0..4, по одному символу на день) разбивается на массив: если `delimeter` не задан — посимвольно (`str_split`), иначе — по разделителю (`explode`). Дальше, поскольку `decode_response` слушатель зарегистрирован пустым (но всё же зарегистрирован — событие "обработано"), базовый `IntegrationDriver::processResponse()` **не** пытается дополнительно применить `json_decode` к результату — ответ возвращается как уже подготовленный `prepare_response`'ом массив строк/символов.

```php
$days = $isdayoff->call('getdata', ['year' => 2026, 'month' => 7]);
// $days — массив однозначных строк на каждый день месяца: '0' (рабочий), '1' (выходной), '2' (сокращённый), и т.д.
```
