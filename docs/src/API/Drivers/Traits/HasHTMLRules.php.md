# HasHTMLRules.php
<!-- DOCGEN:END -->

`trait HasHTMLRules` (`ST_system\API\Drivers\Traits`) — общая логика конвертации произвольного HTML в текстовую разметку, понятную конкретному мессенджеру. Используется бот-драйверами [`MaxBot`](../Bots/MaxBot.php.md) и [`TelegramBot`](../Bots/TelegramBot.php.md) (у `VkBot` HTML-режима нет), каждый из которых определяет свой набор правил через обязательный к переопределению метод `getHtmlRules(): array`.

## Как это работает

1. `normalizeHtml($html)` — точка входа. Принимает либо строку HTML, либо готовый `\DOMDocument`; строка сначала парсится в DOM через `html_to_DOM()` (создаётся `\DOMDocument`, HTML грузится с явным указанием `UTF-8`, ошибки парсинга подавляются через `libxml_use_internal_errors`). Дальше берётся содержимое `<body>` (если DOM-парсер его создал) и рекурсивно обходится через `parse_nodes_recursive()`.
2. `parse_nodes_recursive(\DOMNodeList $nodes)` на каждом узле:
   - **текстовый узел** — схлопывает пробельные последовательности (`preg_replace('/\s+/u', ' ', ...)`) и добавляет в результат, если после `trim()` не пусто;
   - **тег, отсутствующий в карте правил `getHtmlRules()`** — сам тег отбрасывается, но его дочерние узлы обрабатываются рекурсивно (т.е. неизвестная обёртка "прозрачна");
   - **тег с правилом `false`** — вырезается целиком, вместе с содержимым (например `<img>`, `<form>` — в мессенджер их отправить нельзя);
   - **тег с правилом-строкой** — эта строка подставляется вместо тега (например `'br' => "\n"`), без учёта содержимого/дочерних узлов;
   - **тег с правилом-функцией** (`callable`) — вызывается как `$rule(содержимое_детей_после_рекурсивной_обработки, $attributes)`, что позволяет оборачивать/переформатировать содержимое (например обернуть заголовок в `<b>...</b>\n` или превратить `<span class="tg-spoiler">` в `<tg-spoiler>` у Telegram);
   - **тег с правилом `true`** — пропускается "как есть": тег с частью его атрибутов (`href`, `src`, `alt`, `srcset` — единственные переносимые атрибуты, остальные отбрасываются) и рекурсивно обработанным содержимым; `href`/`src`/значения внутри `srcset` дополнительно проходят через `normalize_url()`.
3. Пустые/void-элементы (`area`, `base`, `br`, `col`, `embed`, `hr`, `img`, `input`, `link`, `meta`, `param`, `source`, `track`, `wbr`) при правиле `true` рендерятся как самозакрывающиеся (`<tag ... />`), остальные — как парная пара тегов.

## `normalize_url(string $url)`

Если переданный URL не абсолютный (нет `//` и не начинается с `http`), достраивает его до абсолютного через текущий хост запроса (`$_SERVER['HTTPS']`/`$_SERVER['HTTP_HOST']`) — так относительные ссылки/картинки не ломаются при отправке в мессенджер, где нет контекста текущего сайта.

## `html_to_node_array(string $html)`

Вспомогательный метод — парсит HTML и возвращает плоский PHP-массив дочерних DOM-узлов `<body>` (или корня документа), без рекурсивной конвертации в текст. Полезен, если нужно проинспектировать/обработать узлы верхнего уровня отдельно от полной конвертации через `normalizeHtml()`.

## Как драйверы ботов используют трейт

Каждый драйвер обязан реализовать `protected static function getHtmlRules(): array`, описывающую собственный диалект разметки:

```php
protected static function getHtmlRules(): array {
    $line_break = fn($content) => "$content\n";

    return [
        'br'     => "\n",
        'p'      => $line_break,
        'b'      => true,
        'strong' => true,
        'img'    => false,
        // ...
    ];
}
```

Дальше, в `on_prepare` зарегистрированных HTTP-методов, при определённом условии (у `MaxBot` — `format === 'html'`, у `TelegramBot` — `parse_mode === 'HTML'`) вызывается `self::normalizeHtml($params[$field])`, конвертируя переданный пользователем HTML в разметку, которую понимает API конкретного мессенджера, — до того, как параметры уйдут в HTTP-запрос.
