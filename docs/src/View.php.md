# View.php

# `ST_system\View` — шаблонизатор

Небольшой гибкий шаблонизатор на чистом PHP, по механике похожий на Astro: layout
оборачивает страницу через слот, компоненты получают пропсы, данные текут вниз по дереву
рендера. Есть опциональный донат-кеш с тремя режимами (см. [Кеш](#кеш-донат)).

- Namespace: `ST_system\View` (PSR-4, `ST_system\ => src/`).
- Требует PHP ≥ 7.4. Шаблоны — обычные `.php`-файлы с `<?= ?>` / `<? ?>`.
- Шаблоны зовут голый `View::` — для этого в `init.php` заведён алиас
  `class_alias(ST_system\View::class, 'View')`.

---

## Быстрый старт

**1. Настроить источники (`init.php`):**

```php
use ST_system\View;

class_alias(View::class, 'View'); // чтобы шаблоны звали голый View::

View::setConfig([
    'source' => [
        'layouts'    => '~/storage/layouts',
        'components' => ['source' => '~/storage/components', 'alias' => 'component'],
        'pages'      => ['source' => '~/storage/pages',      'alias' => 'page'],
    ],
]);
```

**2. Написать шаблоны:**

```php
// storage/pages/index.php
View::layouts('main', function () {
    View::component('header');
    ?>
    <main>Привет, <?= htmlspecialchars(View::get('name', 'мир')) ?>!</main>
    <?php
    View::component('footer');
});
```

```php
// storage/layouts/main.php
<!doctype html>
<html lang="ru">
    <? View::slot(['head.title' => 'Заголовок по умолчанию']); ?>
</html>
```

**3. Отрендерить (в контроллере):**

```php
return Response::html(
    (string) View::page('index', ['name' => 'Аня'])
);
```

## Конфигурация

`View::setConfig([...])` (метод из трейта `HasConfig`). Ключи:

| Ключ | По умолчанию | Назначение |
|---|---|---|
| `source` | `[]` | Строка или массив источников (см. [ниже](#источники)) |
| `extension` | `'php'` | Расширение файлов шаблонов (перекрывается per-source) |
| `exclude` | `[]` | Строка или массив исключений (симметрично `source`, побеждает) |
| `cache` | `['use'=>false, 'driver'=>'filesystem', 'dir'=>'']` | Кеш: `use` — режим по умолчанию (обычно off), `driver`/`dir` — где хранить |
| `cascade` | `false` | Каскадировать ли локальный режим/override вглубь потомков (обычно ставится per-call) |

## Источники

Источник маппит **имя-метод** (`page`, `component`, `layouts`, …) на **каталог** шаблонов.
`View::page('index')` ищет `index` внутри каталога источника `page`.

### `source` строкой — автодискавери

```php
View::setConfig(['source' => '~/storage/views']);
```

Каждая папка верхнего уровня становится методом: `views/layouts`, `views/pages` →
`View::layouts()`, `View::pages()`. Имя метода — ровно имя папки (basename).

### `source` массивом — явные имена, алиасы, per-source `extension`

```php
View::setConfig([
    'extension' => 'php',                                    // глобальный дефолт
    'source' => [
        'layouts'    => '~/storage/layouts',
        'components' => ['source' => '~/storage/components', 'alias' => 'component'],
        'pages'      => ['source' => '~/storage/pages',      'alias' => 'page'],
        'icons'      => ['source' => '~/assets/icons', 'extension' => 'svg'], // своё расширение
    ],
]);
```

Ключ регистрирует основное имя, `alias` — дополнительное имя на тот же каталог, `extension` —
расширение только для этого источника (иначе глобальный `extension`).

> `~/` в путях разворачивается в корень документа (`DOCUMENT_ROOT`, иначе `COMPOSER_ROOT`).

### `exclude` — тот же формат, побеждает

`exclude` симметричен `source` по входу (**строка | массив строк**) и на коллизиях **побеждает**.
Каждая запись — либо **путь** (`~/…` или со слэшем → абсолютный префикс), либо **голое имя** (метод):

```php
View::setConfig([
    'source'  => '~/storage/views',
    'exclude' => ['jobs', '~/storage/views/experimental'], // имя-метод + поддерево-префикс
]);
```

- **Имя** (`'jobs'`) — метод/подпапка с таким именем не регистрируется (в т.ч. если явно задан в `source`).
- **Путь-префикс** (`~/storage/views/experimental`) — любой вид, чей файл лежит под этим каталогом,
  недоступен (`View::…('experimental/foo')` → not found), и подпапка-источник под ним не регистрируется.

Резолв ленивый — `exclude` нормализуется один раз, на резолве только дешёвая проверка префикса. Без сканов.

### Резолв путей

`View::page('about')` ищет по порядку и берёт первый существующий (и не исключённый) файл:

1. `<source>/about.php`
2. `<source>/about/index.php`

Вложенность через `/`: `View::page('blog/post')` → `<source>/blog/post.php`. Если ни один не найден —
`RuntimeException`.

Сегмент `..` в имени запрещён: `View::page('../../etc/passwd')` бросит `InvalidArgumentException`
(защита от выхода за пределы каталога, через `Main::preparePath(strict)`). `.` и пустые сегменты
схлопываются безопасно.

## Аргументы фабрики

`View::x(view[, props][, children])`:

- **2-й аргумент**: если это **замыкание** (или не-массив) — это `children` (props пусты); если
  **массив** — это `props`.
- Значит **именованные слоты (массив)** обязаны идти **3-м** аргументом:
  `View::layouts('main', [], ['default'=>fn, 'aside'=>fn])`. Массив 2-м аргументом — всегда props.

```php
View::page('index');                                  // props [], children null
View::page('index', ['name' => 'Аня']);               // props
View::layouts('main', function () { ... });           // children-замыкание (props пусты)
View::layouts('main', ['title'=>'X'], function () {}); // props + children
View::layouts('main', [], ['default'=>fn, 'aside'=>fn]); // props + именованные слоты
```

## Модель рендера

Каждый вызов вида пушит **кадр** в стек рендера. `(string) View::page('index', $props)`:

```
page:index    (storage/pages/index.php)          [index]
 └ layouts:main  (storage/layouts/main.php)       [index, main]
    └ slot() → children()  (замыкание страницы)
       ├ component:header                          [index, main, header]
       ├ ...тело main...
       └ component:footer                          [index, main, footer]
```

Замыкание страницы (`children`) исполняется **лениво — внутри `slot()`**, уже когда layout
рендерится. Поэтому компонент, вложенный в children (например `header`), видит значения,
которые layout только что подставил дефолтом через `slot([...])`.

Ограничение глубины — 50 кадров (самоссылающийся компонент падает внятным исключением).

## Вывод

Любая фабрика возвращает объект `View`. Единственный способ получить HTML — **привести к строке**
(`(string)$view` / `__toString`). Вложенные вызовы печатают себя **сами** — отложенно, при
уничтожении объекта (`__destruct`), поэтому голый `View::component('header');` в шаблоне работает.

```php
// внутри шаблона (вложенный контекст):
View::component('header');                    // временный объект → печатает на ';'
echo View::component('card');                 // __toString → печатает один раз
$html = (string) View::component('card');     // строкой, можно обработать
echo strtoupper($html);
<?= View::component('aside') ?>               // → печатает один раз
```

**Важное правило:** временный объект печатается на `;`, а объект в переменной — когда переменная
умрёт. Присвоил вид в переменную — заверши явным `echo`/`(string)`:

```php
$h = View::component('header'); ?><div>…</div><?php
echo $h;  // иначе header выведется ПОСЛЕ <div>, когда $h выйдет из области видимости
```

> **В контроллере всегда `(string)View::page(...)`.** Печать через авто-эхо внутри вывода
> контроллера стёрлась бы выходным буфером фронт-контроллера. Приведение к строке возвращает HTML,
> который отдаётся в `Response`.

На корне (когда стек кадров снова пуст) приведение к строке автоматически прогоняет
`Assets::finalize()`.

## Пропсы и данные

Данные передаются вторым аргументом фабрики и читаются через `View::get()`.

```php
View::component('header', ['head' => ['title' => 'Главная']]);
// внутри header.php:  View::get('head.title');   // 'Главная'
```

### Точечная нотация

Пропсы нормализуются: вложенный массив и плоский точечный ключ эквивалентны.
`['head' => ['title' => 'X']]` ⇔ `['head.title' => 'X']`; `View::get('head.title')` и
`View::get('head')` работают оба.

### `View::get(string $key, $default = null)`

Ищет по стеку кадров **от текущего вверх к родителям**, затем в глобальном бэге, затем возвращает
`$default`. Ближний кадр выигрывает.

```
View::page('index', ['title' => 'Главная'])
  └ layouts('main')                       // кадр без title
     └ component('header', ['x' => 1])     // кадр с x
        View::get('x')            → header ✓ → 1
        View::get('title')        → header ✗ → main ✗ → index ✓ → 'Главная'
        View::get('missing', '—') → всё ✗ → globals ✗ → '—'
```

> `get()` возвращает **сырое** значение. Экранирование — на совести шаблона:
> `<?= htmlspecialchars(View::get('title'), ENT_QUOTES, 'UTF-8') ?>`.

### `View::set($key, $value = null)` — глобальные значения

Пишет в глобальный бэг (низший приоритет при поиске, виден всем видам в пределах запроса). Принимает
строку+значение или массив:

```php
View::set('brand', 'Whisper');
View::set(['asset' => ['css' => '/public/app.css']]);
View::get('asset.css'); // '/public/app.css' из любого шаблона
```

### `View::name(int $i = 0)` и `View::path(int $i = -1)`

`name()` — короткое имя вида по уровню вложенности; `path()` — путь файла текущего вида (без
расширения), чтобы шаблон знал своё расположение (со-локация ассетов).

```
стек: [page:index, layouts:main, component:header]
View::name()   // 'index'   корень
View::name(-1) // 'header'  текущий
View::path()             // '/…/components/header'  (текущий файл без .php)
dirname(View::path())    // '/…/components'          (своя папка)
View::path().'.'.View::config('extension') // полный путь к файлу
```

## Слоты

`View::layouts('main', $children)` рендерит содержимое страницы **внутрь** layout. Точка вставки —
`View::slot()`. **Формального типа «layout» нет** — вид становится layout'ом ровно тогда, когда его
шаблон зовёт `slot()`.

### Дефолтный слот + fill-семантика

```php
// pages/index.php
View::layouts('main', function () { ?><main>Контент</main><?php });

// layouts/main.php
<html><? View::slot(['head.title' => 'Заголовок по умолчанию']); ?></html>
```

`View::slot($defaults)` по порядку: (1) **ставит дефолты** — только для ключей, которых нет выше по
стеку и в globals (семантика fill, не override, поэтому пропс страницы **побеждает** дефолт layout);
(2) **рендерит children** в этой точке.

### Именованные слоты

```php
// pages/index.php
View::layouts('main', [], [
    'default' => function () { ?><main>основной</main><?php },
    'aside'   => function () { ?><aside>сайдбар</aside><?php },
]);
// layouts/main.php
<div><? View::slot(); ?></div><aside><? View::slot('aside'); ?></aside>
```

| Форма | Что делает |
|---|---|
| `slot()` | Дефолтный слот |
| `slot(array $defaults)` | Дефолты + дефолтный слот |
| `slot(string $name)` | Именованный слот |
| `slot(string $name, array $defaults)` | Дефолты + именованный слот |

### Пограничные случаи (мягкая деградация)

- **`slot()` без детей** (вид вызван без замыкания): дефолты проставляются, контент рисуется **пусто** — без ошибки.
- **Массив параметров в «layout»** (`View::layouts('main', ['title'=>'X'])`): 2-й арг-массив — это **props**, `children=null` → слоты пусты, props доступны.
- **Именованные слоты 2-м аргументом**: массив трактуется как **props** — слоты будут пустыми. Передавай их **3-м** аргументом.
- **`children` в вид без `slot()`**: замыкание никогда не вызовется — тихо **игнорируется**.
- **`slot()` вне рендера**: единственная жёсткая ошибка — `LogicException`.

## `View::capture(callable $fn): string`

Захватывает эхо-вывод замыкания в строку — когда нужно собрать несколько печатающих вызовов разом.
**Не трогает стек кадров**, поэтому `View::get()` внутри видит те же кадры, что и снаружи. Это не про
конфиг — отдельный инструмент.

```php
$html = View::capture(function () {
    View::component('a');
    View::component('b');
});
```

## Кеш (донат)

Кеш **opt-in** — по умолчанию вид не кешируется (рендерится на каждый запрос). Включается тумблером
`->cache(...)` на конкретном вызове. Кеширующий вид («граница») хранит свой **скелет**.

### Три режима

| Вызов | Родитель | Дети |
|---|---|---|
| `->cache('full')` — **монолит** | кешируется, **если всё поддерево заморозимо** (иначе граница не кешируется — см. [Что кладётся в кеш](#что-кладётся-в-кеш-props-дыр-и-sets)) | **всё поддерево запекается** в один блоб (режимы детей игнорируются); dep-stamp = свой файл + все вложенные; правка любого → пересбор |
| `->cache(true)` — **глубокий (донат)** | кеширует свой скелет | **все дети — дыры**, каждый по своему режиму; dep-stamp = только свой файл, дети инвалидируются сами |
| `->cache(false)` — **живой** | не кешируется | сам живая дыра в кеширующем родителе / обычный рендер на корне |

`->cache()` без аргумента = `->cache(true)`. Дефолт (без тумблера) — не кешируется.

Внутри **глубокого** родителя ребёнок ведёт себя по своему режиму:
- ребёнок `->cache(true)`/`->cache('full')` → своя граница (свой кеш, инвалидируется независимо);
- ребёнок `->cache(false)` или без тумблера → **живая дыра** (ре-рендер на каждый хит родителя).

```php
// components/product-card.php  — вызвана как ->cache(true)
<article>
  <img src="<?= e(View::get('img')) ?>">                // статика → своя граница/дыра по режиму
  <h2><?= e(View::get('title')) ?></h2>
  <?php View::component('cart-count')->cache(false); ?>  // ЖИВАЯ дыра (свежая каждый хит)
  <?php View::component('reviews', ['id'=>View::get('id')])->cache(true); ?> // СВОЯ граница
</article>
```

- Правка `reviews.php` → пересоберётся только кеш reviews, скелет карточки цел.
- Правка `product-card.php` → пересоберётся скелет карточки.
- `cart-count` свежий всегда.
- `->cache('full')` на карточке заморозил бы всё поддерево в один блоб (никаких дыр, режимы детей игнор).

### `->cascade(bool $on = true)` — режим вглубь

Пробрасывает локальный режим/override на всё поддерево (пока потомок не выставит своё). Полезно
включить/выключить кеш целому региону: `->cache(false)->cascade()` — весь регион живой;
`->cache(true)->cascade()` — каждый вид поддерева кеширует свой скелет.

### Инвалидация

Без TTL: кеш «вечный», инвалидируется только по изменениям:
- **dep-stamp** — mtime файла границы (в `'full'` — плюс всех запечённых потомков);
- **reads-stamp** — значения чужого контекста, которые вид прочитал через `View::get()` (см. ниже).

### Корректность при чтении/записи контекста

Кешируемый вид отдаётся по ключу (имя + свои props). Но `View::get()` читает и **чужое** (пропсы
предков, globals), а `View::set()` пишет globals. На хите шаблон границы не выполняется, поэтому:

- **`reads`** — вид запоминает, что он прочитал через `View::get()` из чужого; на проверке эти ключи
  перечитываются из текущего контекста и сверяются. Изменилось → пересборка. *(Shared-компонент вроде
  `header`, читающий `View::get('head.title')` страницы, на другой странице корректно пересоберётся, а
  не застрянет.)*
- **`sets`** — вид запоминает свои `View::set()`; на хите они **проигрываются** обратно в globals
  перед заполнением дыр, чтобы дети видели вычисленные значения.

Оба механизма автоматические — кешировать можно виды, читающие/пишущие контекст, без ручной «чистоты».

### Что кладётся в кеш (props дыр и sets)

Скелет границы хранит свои **дыры** (props/children каждой отложенной дыры) и **sets** — они
персистятся нативным `serialize()` и восстанавливаются на каждом хите. Поэтому:

- **Объекты переживают round-trip по значению** — класс и приватные поля сохраняются (нативный
  `serialize()` честнее, чем `jsonSerialize()`), включая enum, `DateTime` и циклические графы.
  Класс объекта должен быть автозагружаем на момент чтения кеша.
- **Замыкание/ресурс на входе дыры → граница НЕ кешируется** (poison). Такой вход требует живого
  ре-рендера, а запечь живого ребёнка родитель не может — поэтому **ближайшая кеш-граница
  становится некешируемой и перерисовывается на каждый хит** (замыкание используется свежим).
  Живость **всплывает вверх**: если у самой границы props сериализуемы, родитель держит её живой
  **дырой** (`->cache(true)`) и продолжает кешироваться сам; иначе отравляется и родитель.
  **На других потомков не влияет** — потомок со своим `->cache()` кешируется независимо (poison
  ограничен ближайшей границей). `View::set()` замыкания/ресурса действует так же (граница не
  кешируется, без ошибки). Нужна кешируемая граница с «поведением» — передавай инвокабл-объект/
  `[Class,'method']` (сериализуются) или тяни данные через контекст (`View::get`).
- **`->cache('full')`** валиден, только если **всё** поддерево заморозимо; ребёнок с незаморозимым
  входом (замыкание/ресурс в props) отравляет и full-границу — она рендерится живой, а не монолитом.

> **Что НЕ отслеживается:** внешнее состояние вне `View::get()` (время `date()`, сессия, БД, random).
> Вид, который в `View::set()` или вывод тянет такое, при кешировании застынет — не кешируй его
> (`->cache(false)`), или прими это. Аналогично сайд-эффекты `Assets::add()` из кешируемой границы на хите
> не воспроизводятся — не кешируй виды, которые их дергают (или выноси ассеты в живую дыру).

## Интеграция с `ST_system\Assets`

`View` владеет выводом, поэтому `Assets` работает в **строковом** режиме. В `init.php`:

```php
use ST_system\Assets;
Assets::setConfig(['bufferization' => false]);
```

- в шаблоне ставится точка сборки: `Assets::mount('head')` (печатает плейсхолдер);
- любой вложенный вид добавляет ресурсы: `Assets::addString(...)`, `Assets::addCss(...)` в буфер;
- корневой `(string)$view` автоматически прогоняет `Assets::finalize()` — плейсхолдер заменяется.

> `Assets::add*()` — сайд-эффект: из **кешируемой** границы на хите он не воспроизводится (см. каveat
> в разделе Кеш). Ставь `Assets::add*()` в живых дырах или в некешируемых видах.

## Полный пример

```php
// init.php
class_alias(ST_system\View::class, 'View');
View::setConfig([
    'source' => [
        'layouts'    => '~/storage/layouts',
        'components' => ['source' => '~/storage/components', 'alias' => 'component'],
        'pages'      => ['source' => '~/storage/pages',      'alias' => 'page'],
    ],
    'cache' => ['use' => false, 'driver' => 'filesystem', 'dir' => '~/cache/views'],
]);
```

```php
// Api/Controllers/PageController.php
public function index(): Response {
    return Response::html(
        (string) View::page('index', ['head' => ['title' => 'Главная']])
    );
}
```

```php
// storage/pages/index.php
View::layouts('main', function () {
    View::component('header');
    ?><main><h1><?= htmlspecialchars(View::get('head.title')) ?></h1></main><?php
    View::component('footer');
});
```

## Справочник публичного API

### Конфигурация (из `HasConfig`)

| Метод | Описание |
|---|---|
| `View::setConfig(array $config): void` | `source` / `extension` / `exclude` / `cache` / `cascade` |
| `View::config(string $key = '')` | Прочитать конфиг по точечному ключу |

### Фабрики видов (динамические имена из `source`)

| Вызов | Возвращает |
|---|---|
| `View::page(string $name, array $props = [], $children = null): View` | Вид страницы |
| `View::layouts(string $name, $children): View` | Layout (2-й арг — замыкание/массив слотов) |
| `View::component(string $name, array $props = []): View` | Компонент |

> Имена методов берутся из `source`/`alias`. Аргументы: 2-й — `children` если замыкание/не-массив,
> иначе `props`; именованные слоты (массив) — 3-м аргументом.

### Статические помощники

| Метод | Описание |
|---|---|
| `View::get(string $key, $default = null)` | Сырое значение: кадр → родители → globals → default |
| `View::set($key, $value = null): void` | Записать в globals |
| `View::name(int $i = 0): string` | Короткое имя вида по уровню (0 — корень, отрицательный — от текущего) |
| `View::path(int $i = -1): string` | Путь файла текущего вида без расширения (−1 — текущий) |
| `View::slot(...$args): void` | Отрендерить слот (+ поставить дефолты) |
| `View::capture(callable $fn): string` | Захватить эхо-вывод в строку |

### Методы экземпляра

| Метод | Описание |
|---|---|
| `(string)$view` / `->__toString()` | HTML строкой; на корне финализирует Assets. Повторный вызов → `''` |
| `->cache($mode = true): self` | Режим кеша: `'full'` монолит / `true` глубокий / `false` живая дыра (не кешируется) |
| `->cascade(bool $on = true): self` | Пробросить режим/override вглубь потомков |

Тумблеры вызываются **до** рендера и чейнятся: `View::component('x', $props)->cache(true)->cascade()`.

### Зарезервированные имена

Источник не может называться так же, как метод: `get`, `set`, `slot`, `capture`, `config`,
`setConfig`, `applyConfig`, `sources`, `name`, `path`, `cache`, `cascade` — иначе
`LogicException` при резолве источников.

## Подводные камни

- **Контроллер → `(string)View::page(...)`** — иначе авто-эхо сотрёт выходной буфер фронт-контроллера.
- **Вид в переменной печатается при её уничтожении.** Присвоил — заверши явным `echo`/`(string)`.
- **`get()` не экранирует.** Экранируйте в шаблоне (`htmlspecialchars`), особенно пользовательский ввод.
- **`slot([...])` ставит дефолты, а не перезаписывает** — пропсы страницы всегда сильнее дефолтов layout.
- **Именованные слоты — только 3-м аргументом** (массив 2-м — это props).
- **Кеш — opt-in.** Мелочь не кешируй (оверхед > рендера); кешируй тяжёлые/стабильные виды. `reads`/`sets`
  делают их корректными при чтении/записи контекста, но внешнее состояние (`date`, сессия, `Assets::add`)
  не отслеживается — такие виды держи живыми (`->cache(false)`).
