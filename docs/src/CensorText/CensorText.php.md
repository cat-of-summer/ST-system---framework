# CensorText.php
<!-- DOCGEN:END -->

`class CensorText` (`ST_system\CensorText`) — фильтр нецензурной/нежелательной лексики по словарю с нормализацией текста (нижний регистр + опциональная leetspeak-подстановка похожих символов) и внутренним per-instance кешем результатов по MD5 текста. Самодостаточный класс, не зависит от других частей фреймворка — конструируется напрямую со списком «плохих слов», сгруппированным по категориям.

## Конструктор

```php
public function __construct(array $bad_words, bool $use_normalization = true)
```

- `$bad_words` — ассоциативный массив `группа => список слов-корней`:
  ```php
  $censor = new CensorText([
      'mat'     => ['слово1', 'слово2'],
      'insults' => ['дурак', 'идиот'],
  ]);
  ```
- `$use_normalization` — включает ли `normalize()` замену символов по `$normalization_map` (по умолчанию `true`).

## normalization_map_add(array $map): void

Домешивает дополнительные пары `символ => замена` в карту нормализации. По умолчанию карта уже содержит самые частые leetspeak-подстановки под кириллицу: `6`→`б`, `@`→`а`, `3`→`з`, `0`→`о`, `4`→`ч`, `1`/`!`→`и`, `$`→`с`, `*`/`.`/`,` → удаление, `ё`→`е`. Новые записи имеют приоритет — `array_merge` перезаписывает совпадающие ключи.

```php
$censor->normalization_map_add(['5' => 'ы']);
```

## check(string $text): array

Проверяет текст на совпадение с каждой группой слов независимо и возвращает `[группа => bool]`. Слово считается найденным по паттерну `/\b<корень>\w*/ui`, то есть подходит любое слово, начинающееся с указанного корня (учитываются словоформы/окончания). Результат кешируется по `md5($text)` внутри инстанса — повторный вызов с тем же текстом не пересчитывается.

```php
$censor->check('ты дурак!');
// ['mat' => false, 'insults' => true]
```

## checkAll(string $text): bool

`true`, если хотя бы одна группа из `check()` дала совпадение — быстрая проверка «текст плохой хоть чем-то».

```php
if ($censor->checkAll($text)) {
    // текст содержит запрещённую лексику
}
```

## count(string $text): array

Считает число совпадений по каждой группе (`preg_match_all` по каждому слову-корню, суммируется по группе), но включает в результат **только** группы с хотя бы одним совпадением — группы без совпадений в массиве отсутствуют (а не равны `0`). Кешируется отдельно от `check()`/`censor()` (свой под-ключ в кеше инстанса по тому же `md5($text)`).

```php
$censor->count('дурак дурак идиот');
// ['insults' => 3]
```

## countAll(string $text): int

`array_sum($this->count($text))` — суммарное число всех совпадений по всем группам.

## censor(string $text): string

Возвращает **нормализованную** версию текста (нижний регистр + leetspeak-замены, если включены), в которой каждое найденное слово (вместе со всем «хвостом», подошедшим под `\w*`) заменено на `*` той же длины в символах (`mb_strlen`). Возвращается именно нормализованный текст, а не оригинал с точечными заменами, — регистр и leetspeak-символы в результате уже не совпадают с исходным вводом.

```php
$censor->censor('Ты ДУРАК!!!');
// 'ты *****!!!'
```

## Нормализация и кеш (внутреннее устройство)

`normalize()` — приватный метод, общий для `check()`, `count()` и `censor()`: приводит текст к нижнему регистру (`mb_strtolower`), затем, если `$use_normalization` включена, применяет `strtr()` с `$normalization_map`. Подстановка выполняется **после** lowercase, поэтому в `normalization_map` достаточно указывать только строчные ключи.

Все три проверочных метода используют общий `$this->cache[md5($text)]`, но с разными под-ключами (`check`, `count`, `censor`) — один и тот же текст, проверенный всеми тремя способами, вычисляется по одному разу на способ. Кеш живёт только в памяти в рамках жизни инстанса, без TTL и ограничения размера.
