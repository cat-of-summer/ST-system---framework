# HasInstance.php
<!-- DOCGEN:END -->

## Назначение

`HasInstance` — минимальный трейт синглтона: единственный статический метод, который лениво создаёт и кэширует единственный экземпляр подмешивающего класса. Используется классами, которые хотят иметь внутреннее состояние (например, кэш соединений, накопленные слушатели событий), оставаясь при этом со статическим/фасадным публичным API. Во фреймворке им пользуются `Access` и `Debug` — оба реализуют статический интерфейс поверх приватного `__construct()` и приватного экземпляра.

## Что добавляет подмешивающему классу

### `private static function getInstance(...$args): self`

Возвращает единственный экземпляр класса:

- при первом вызове создаёт `new static(...$args)` и запоминает его в статической переменной, локальной для метода (`static $instance = null;`);
- при всех последующих вызовах возвращает уже созданный экземпляр, **игнорируя** любые `$args`, переданные повторно (они учитываются только при самом первом вызове, когда `$instance === null`).

Поскольку переменная `$instance` объявлена как `static` внутри самого метода трейта, а не как свойство класса, у **каждого класса**, подмешивающего `HasInstance`, будет собственный независимый экземпляр — благодаря `new static(...)` и тому, что PHP поддерживает отдельную копию статической переменной функции на каждый вызывающий класс через позднее статическое связывание.

Метод объявлен `private`, поэтому напрямую извне не вызывается — подмешивающий класс должен сам открыть к нему доступ там, где это нужно, либо использовать его только внутри своих же статических методов.

## Практический пример

```php
final class Access {
    use HasInstance;

    private function __construct() {
        // тяжёлая инициализация (кэш, конфигурация) — выполнится один раз
        $this->cache = CacheManager::make(...);
    }

    public static function handleIp(): void {
        $self = self::getInstance(); // тот же самый объект при каждом вызове
        // ...
        $self->fire('ban', $ip);
    }
}
```

`Debug` использует ровно тот же паттерн — приватный конструктор, `getInstance()` только для внутреннего использования статическими методами-фасадами (`Debug::backtrace()`, `Debug::handleError()` и т.д. обращаются к `static::getInstance()` там, где нужен доступ к нестатическим методам/трейтам вроде `HasEvents`).

## Нюансы

- Не является потокобезопасным (во фреймворке это неактуально — классический PHP-процесс на запрос).
- Не даёт способа сбросить/подменить синглтон — если это нужно (например, в тестах), это ответственность самого класса (трейт такого метода не предоставляет).
- Конструктор подмешивающего класса обычно делают `private`, чтобы гарантировать создание экземпляра только через `getInstance()`.
