# HasEvents.php
<!-- DOCGEN:END -->

## Назначение

`HasEvents` даёт классу минимальный pub/sub-механизм: возможность подписаться на именованное событие (`on()`) и уведомить всех подписчиков (`fire()`/`trigger()`). Слушатели хранятся в **состоянии инстанса**, поэтому трейт для классов, у которых есть настоящий объект: `HTTP\WebClient`, `API\IntegrationDriver`, большинство `API\Drivers\*`.

Для классов-**фасадов** без инстанса (`Access`, `Debug`, `Lang`, `View`) есть статический близнец — [`HasStaticEvents`](HasStaticEvents.php.md): тот же API, но слушатели на класс. Раньше фасады эмулировали статические события через `getInstance()->_on()`-обёртку над `HasEvents`; этот костыль убран.

## Что добавляет подмешивающему классу

### `private array $listeners`

Внутреннее хранилище: `[имя_события => [callable, callable, ...]]`. Приватное — не видно и не переопределяется наследниками напрямую.

### `protected static function getReservedEvents(): array`

Точка расширения (по умолчанию `[]`). Возвращает список имён событий, которые считаются "внутренними" и не могут быть вызваны извне через `trigger()` — только самим классом через `fire()`. Пример (`Debug.php`):

```php
protected static function getReservedEvents(): array {
    return ['on_error', 'on_exception', 'on_shutdown'];
}
```

### `public function on(string $event, callable $listener): void`

Регистрирует слушателя для события `$event`. Можно зарегистрировать сколько угодно слушателей на одно и то же событие — вызываются все, в порядке регистрации.

`on()` — инстанс-метод: подписка идёт на конкретный объект (`$client->on('response', ...)`). Классу-фасаду без инстанса нужен статический вариант — тогда берут [`HasStaticEvents`](HasStaticEvents.php.md), а не обёртку над синглтоном.

### `protected function fire(string $event, &...$params)`

Вызывает всех зарегистрированных слушателей события `$event`, передавая им `$params` **по ссылке** (сигнатура `&...$params` — variadic-by-reference, поэтому вызывающий может рассчитывать, что слушатель способен изменить переданные аргументы).

Ключевая семантика: **если для события нет ни одного слушателя, `fire()` возвращает `false` и не вызывает `call_user_func_array`.** Если слушатели есть, метод после цикла foreach ничего явно не возвращает (`return` без значения => `null`). Это используется по всему фреймворку как флаг "было ли событие кем-то обработано" — обычно в паре с fallback-поведением по умолчанию:

```php
// на инстансе (HasEvents):
if ($this->fire('decode_response', $raw, $result) === false)
    $result = json_decode($raw, true);
```

То есть: если хотя бы один слушатель зарегистрирован — фреймворк доверяет ему обработку и не выполняет поведение по умолчанию; если слушателей нет — срабатывает дефолт. Идиома одинакова у обоих трейтов; в статических фасадах (`Access::throw()`, `Debug::onError()`) она же, но через `self::fire(...)` из [`HasStaticEvents`](HasStaticEvents.php.md).

Важно: сравнение именно `=== false` (строгое), поскольку слушатель может ничего не возвращать (тоже `null`), и это не должно ложно трактоваться как "не обработано".

### `public function trigger(string $event, &...$params)`

Публичная, "безопасная" версия `fire()` для внешних вызывающих: перед вызовом проверяет, что `$event` не входит в `getReservedEvents()` — если входит, бросает `\LogicException`. Иначе делегирует в `fire()` и возвращает то же значение (включая `false` при отсутствии слушателей).

## Нюансы

- `fire()` — `protected`, вызывается только изнутри самого класса (обычно в момент наступления соответствующего внутреннего события).
- `trigger()` — `public`, предназначен для внешнего кода, который хочет искусственно инициировать событие, но не должен иметь возможность подделать зарезервированные системные события.
- Слушатели вызываются синхронно и в порядке регистрации; исключение в одном слушателе прервёт обход оставшихся (никакого перехвата ошибок трейт не делает).
- `$listeners` не сбрасывается никаким методом трейта — если нужен сброс подписок, это забота подмешивающего класса.
