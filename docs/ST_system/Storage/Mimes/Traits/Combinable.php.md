# Combinable.php

> Документация сгенерирована AI-агентом на основе исходного кода.

## 1. Концепция

Трейт для MIME-сервисов, которые умеют объединять несколько файлов одного типа в один (CSS, JS, шрифты). Результат кэшируется на диске — повторный вызов `combine()` с тем же набором файлов возвращает кэш без пересчёта, если ни один из исходников не изменился.

Кэш-ключ формируется как md5 от списка абсолютных путей. Пересчёт происходит если:
- `mtime` хотя бы одного исходника новее `mtime` кэш-файла, или
- передан `['force' => true]`

```php
$css = File::make('~/assets/app.css');          // CssMime
$combined = $css->combine([
    '~/assets/reset.css',
    '~/assets/app.css',
]);

// Принудительная перегенерация
$combined = $css->combine($files, ['force' => true]);

// Часто используется вместе с minify()
$combined = $css->combine($files)->minify();
```

## 2. Публичный метод

### `combine($files, array $config = []): File`

Принимает один файл или список файлов любой из форм:

| Тип элемента | Поведение |
|---|---|
| `File` | Передаётся как есть |
| URL-строка | Оборачивается в `File::make($url)` |
| Путь-строка | Ищется через `File::find()`, fallback — `File::make($path)` |

Возвращает `File` с указателем на объединённый кэш-файл.

**Параметр `$config`:**

| Ключ | Тип | Описание |
|---|---|---|
| `force` | `bool` | Принудительно пересоздать кэш, даже если он актуален |

## 3. Защищённый метод

### `__combineExtension(): string`

Возвращает расширение выходного файла. По умолчанию — расширение исходного файла (`$this->file->getExtension()`). Классы-потребители могут переопределить, если нужно другое расширение.

## 4. Абстрактный метод

### `abstract __combine(array $files, array $config): string`

Должен быть реализован в использующем трейт классе. Принимает уже разрезолвленный массив `File[]` и конфиг, возвращает строку с объединённым содержимым.

```php
// Пример реализации в CssMime
protected function __combine(array $files, array $config): string {
    return implode("\n", array_map(fn($f) => $f->getContents(), $files));
}
```

## 5. Разрешение входных данных

Перед объединением все элементы проходят через приватный `__resolveFiles()`:

1. Если элемент — `File`: используется напрямую.
2. Если строка пустая: пропускается.
3. Если строка — URL (`filter_var(..., FILTER_VALIDATE_URL)`): `File::make($url)`.
4. Иначе — поиск через `File::find()` (по uri или относительно текущего файла). Если не найдено — `File::make($path)`.
